# 本地联调检查清单

## 前置条件

两个终端窗口都要运行：

| 终端 | 命令 | 预期输出 |
|------|------|---------|
| 终端 1 (后端) | `cd server && npm run dev` | `Server running on http://localhost:3001` |
| 终端 2 (前端) | `cd web && npm run dev` | `http://localhost:5173` |

---

## 检查步骤

### 1. 健康检查

```bash
curl http://localhost:3001/api/health
```

**预期结果：**
```json
{"status":"ok","timestamp":"..."}
```

---

### 2. 测试数据存在

在浏览器打开 http://localhost:5173

**预期结果：**
- 页面顶部有深色导航栏："科研视觉案例库"
- 显示 3 张卡片：
  - 荧光显微镜下的神经元细胞（显微图/生命科学）

点击导航栏的"导出"：

### 8. API 直接测试

```bash
# 列出所有案例
curl http://localhost:3001/api/cases

# 查看某个案例详情（替换 ID）
curl http://localhost:3001/api/cases/{id}

# 提交一个新案例（从 URL 下载图片）
curl -X POST http://localhost:3001/api/captures \
  -F "source_url=https://example.com/test" \
  -F "page_title=API测试" \
  -F "image_url=https://picsum.photos/seed/test/800/600" \
  -F "context_text=API直接测试" \
  -F "capture_type=image"

# 确认图片已保存（查看返回的 imagePath）
# 浏览器打开 http://localhost:3001{imagePath} 确认图片可访问

# 导出 CSV
curl http://localhost:3001/api/export/csv
```

### 9. Chrome 插件（需要先加载）

**加载插件：**
1. Chrome 地址栏输入 `chrome://extensions`
2. 右上角打开"开发者模式"
3. 左上角"加载已解压的扩展程序"
4. 选择 `extension/` 文件夹

**检查点：**

| 测试 | 操作 | 预期结果 |
|------|------|---------|
| 连接状态 | 点击插件图标 | 显示"后端连接正常"（绿色圆点） |
| 截图保存 | 任意网页 → 点击插件 → "保存当前页面截图" | 弹出"保存成功" |
| 图片保存 | 任意网页 → 点击插件 → "保存当前页面主要图片" | 弹出"保存成功" |
| 右键图片 | 右键图片 → "保存到科研视觉案例库" | 右上角弹出通知"保存成功" |
| 选中文字 | 选中文字 → 右键 → "保存选中内容" | 右上角弹出通知"保存成功" |
| 地址配置 | 插件弹出框修改 API 地址为 `http://localhost:3001` | 状态变为"后端连接正常" |

## 常见问题排查

| 问题 | 原因 | 解决 |
|------|------|------|
| 前端显示空白 | 后端没启动 | 检查终端 1 是否在运行 |
| 图片不显示（裂图） | 种子数据无真实图片 | 属于正常情况，新采集的图片会正常显示 |
| 插件连不上后端 | API 地址错误 | 插件弹出框中输入 `http://localhost:3001` |
| 保存失败 | 后端未运行 | 先启动后端 |
| 筛选没反应 | 前端代理问题 | 刷新页面重试 |
