## 第一部分：三轴选项字典

三轴分类法是贯穿整个项目的上层分析框架，回答三个独立问题：
- **功能用途**：影像为何存在？（Why）
- **传播媒介**：影像如何被观看与传播？（How-Distribute）
- **技术手段**：影像如何被生产？（How-Produce）

每个轴有6个互斥（单个案例可多选）的主类，各含精确定义、边界说明、典型案例。

---

### 轴一：功能用途（Why）

影像的根本目的——它为什么被创造、承载什么功能。

#### 1.1 记录（Record）

| 属性 | 内容 |
|------|------|
| **精确定义** | 以客观呈现现实存在为核心目的的影像，强调忠实还原对象原貌，非主观阐释。 |
| **属于** | 实验室现场照片、实验过程抓拍、人物肖像、设备外观、场地环境记录、病理切片存档。 |
| **不属于** | 附带文字标注解释机制的示意图（→解释）、展示成果美感的精修产品图（→展示）、经过艺术化处理的封面（→传播）。 |
| **边界区分** | 与"展示"的边界：记录强调客观性、存档性，展示强调美感、提升印象。一张未经修饰的实验室随手拍是记录，同一场景经专业灯光和构图后用于官网首页则偏向展示。 |
| **典型案例1** | MIT实验室页面中科研人员操作仪器的纪实照片——无摆拍痕迹，保留原始光线和构图。 |
| **典型案例2** | 论文中"Figure S1: Experimental setup"的实验装置现场照片——未加标注，仅呈现装置原貌。 |
| **典型案例3** | 科研团队合影——在实验室或办公楼前拍摄，用于官网团队介绍页，无过度修图。 |

#### 1.2 解释（Explain）

| 属性 | 内容 |
|------|------|
| **精确定义** | 以阐明机制、结构、流程或原理为核心目的的影像，通常含标注、箭头、文字说明。 |
| **属于** | 信号通路图、细胞结构示意图、设备工作原理图、流程图、组织结构层级图。 |
| **不属于** | 无标注的3D渲染图仅做美化展示（→展示）、纯数据趋势图不做机制说明（→数据）、AI生成的装饰性概念图不含教学意图（→传播）。 |
| **边界区分** | 与"数据"的边界：解释是对"为什么/如何"的回答，数据是对"是什么/多少"的呈现。一个带标注的细胞分裂过程图是解释，一个Western blot结果条带图是数据。 |
| **典型案例1** | Cell论文中带箭头标注的信号通路机制图——逐步展示受体激活→磷酸化→转录调控全过程。 |
| **典型案例2** | Nature Reviews的综述示意图——用插画方式展示某疾病的病理机制。 |
| **典型案例3** | 工科论文中设备结构剖面图——标注各部件名称与功能。 |

#### 1.3 数据（Data）

| 属性 | 内容 |
|------|------|
| **精确定义** | 以呈现量化信息或结构化数据为核心的影像，包括统计图表、模型输出、趋势曲线。 |
| **属于** | 柱状图/折线图/散点图、热图、Western blot条带图、流式细胞图、FTIR/XRD图谱、统计表。 |
| **不属于** | 用小图标装饰但核心是解释概念的图（→解释）、用3D渲染呈现的数据可视化（→技术手段=渲染）。 |
| **边界区分** | 与"解释"的边界：数据的核心是数字/量/关系，解释的核心是机制/原理/过程。与"展示"的边界：数据强调信息准确传递，展示强调视觉冲击。 |
| **典型案例1** | 论文中比较三组药物疗效的柱状图+误差线——呈现实验结果。 |
| **典型案例2** | Nature论文中展示全球碳排放趋势的折线图——呈现时序数据。 |
| **典型案例3** | 蛋白质组学论文中的火山图（Volcano plot）——同时呈现差异显著性和表达倍数。 |

#### 1.4 展示（Showcase）

| 属性 | 内容 |
|------|------|
| **精确定义** | 以塑造正面印象、突出成果价值或美化呈现为核心的影像，强调视觉品质感。 |
| **属于** | 产品精修渲染图、成果海报、获奖项目配图、空间设计效果图、机构形象照。 |
| **不属于** | 客观记录的设备照片无美化意图（→记录）、带详细机制标注的示意图（→解释）、纯数据呈现（→数据）。 |
| **边界区分** | 与"传播"的边界：展示面向专业/内部受众、目标是建立专业形象；传播面向公众/外部、目标是扩散影响。同一张图片放在论文插图里是展示，放在微博科普帖里是传播。 |
| **典型案例1** | 追觅科技官网上产品的高品质3D渲染图——展示马达结构、材质质感。 |
| **典型案例2** | 论文的Graphical Abstract——用简洁图展示研究核心发现，提升论文可见度。 |
| **典型案例3** | 科研机构的项目成果海报——用图片+关键数据展示项目亮点和影响力。 |

#### 1.5 传播（Outreach）

| 属性 | 内容 |
|------|------|
| **精确定义** | 以面向公众或外部受众扩散信息、塑造品牌或普及知识为核心的影像。 |
| **属于** | 期刊封面、科普图解、品牌宣传图、招生海报、社交媒体传播图、新闻配图。 |
| **不属于** | 论文内用于解释机制的插图（→解释）、仅供内部存档的实验记录（→记录）。 |
| **边界区分** | 与"解释"的边界：传播的核心是吸引关注、易于理解和分享；解释的核心是准确阐明机制。Nature封面上一个艺术化处理的蛋白结构图是传播，同一蛋白的论文内机制图是解释。 |
| **典型案例1** | Nature封面——将当期亮点研究转化为具有视觉冲击力的艺术化图像。 |
| **典型案例2** | NASA社交媒体发布的"今日天文照片"——附说明面向公众科普传播。 |
| **典型案例3** | 大学招生手册中的校园与实验场景图——面向潜在学生传播品牌形象。 |

#### 1.6 交互（Interact）

| 属性 | 内容 |
|------|------|
| **精确定义** | 以支持用户操作、浏览、筛选或探索为核心的影像/界面，强调人机互动性。 |
| **属于** | 交互地图、可缩放图像、筛选仪表盘、3D模型浏览器、数据探索面板。 |
| **不属于** | 静态截图展示交互界面但不可操作（→传播或展示）、论文中的静态图表（→数据）。 |
| **边界区分** | 与"数据"的边界：交互的核心是用户可以操控视图变换，静态数据图只能被动观看。一个在线COVID-19仪表盘（可筛选国家/时间）是交互，同样数据的截图放入论文是数据。 |
| **典型案例1** | Johns Hopkins大学COVID-19疫情追踪仪表盘——可交互筛选国家、时间线、数据维度。 |
| **典型案例2** | Allen Institute的脑图谱3D浏览器——用户可旋转、缩放、点击不同脑区查看细胞类型。 |
| **典型案例3** | 蛋白质数据库(PDB)的3D分子浏览器——用户可旋转、测量、标记分子结构。 |

### 轴二：传播媒介（How-Distribute）

影像被观看、传播的物质与技术载体。

#### 2.1 静图（Static Image）

| 属性 | 内容 |
|------|------|
| **精确定义** | 单张静止、不可交互的图像文件，通过屏幕或印刷显示。 |
| **属于** | JPG/PNG/TIFF格式照片、网页上的单张插图、论文中的单图、SVG矢量图、PDF中的图片页。 |
| **不属于** | GIF动图（→动图）、多张图组合成一个页面的版面（→图组）、嵌入网页的交互可视化（→交互）。 |
| **边界区分** | 与"图组"的边界：静图是单张独立图像；图组是多张图片以版面方式组合。同一篇论文的Figure 1a（单张）是静图，Figure 1的全部子图拼合版面是图组。 |
| **典型案例1** | Nature论文中单独编号的Figure 1a——一张细胞荧光显微照片。 |
| **典型案例2** | 科研团队官网上展示研究成果的单张JPG图片。 |
| **典型案例3** | Science期刊电子版中嵌入的SVG科学插图。 |

#### 2.2 动图（Animated Image）

| 属性 | 内容 |
|------|------|
| **精确定义** | 以循环播放为核心的短动画，无需用户控制播放，通常无音频。 |
| **属于** | GIF动图、短循环动效（如APNG、WebP动画）、论文补充材料中的动态演示。 |
| **不属于** | 有音频和播放控制的视频（→视频）、需要用户拖拽操作的交互模型（→交互）。 |
| **边界区分** | 与"视频"的边界：动图强调自动循环、无音频、通常<10秒；视频有播放控制、常含音频、时长不定。 |
| **典型案例1** | 论文补充材料中展示蛋白质构象变化的GIF动图——自动循环播放。 |
| **典型案例2** | 科普推文中用于解释科学原理的短循环动效——如日食形成过程的GIF。 |
| **典型案例3** | Chemical Science期刊电子版中展示分子旋转的循环动画。 |

#### 2.3 视频（Video）

| 属性 | 内容 |
|------|------|
| **精确定义** | 有时序播放、可控制（暂停/进度条）、通常含音频的动态影像。 |
| **属于** | 实验室Vlog、科研成果讲解视频、论文视频摘要、纪录片、MOOC课程视频。 |
| **不属于** | 无声的GIF循环动图（→动图）、可交互的3D场景（→交互）。 |
| **边界区分** | 与"动图"的边界：视频有播放控件、通常含音频、时长更长。与"交互"的边界：视频是线性播放，交互是用户自由操控视角/数据。 |
| **典型案例1** | Nature Video频道中介绍某篇论文的3分钟讲解视频。 |
| **典型案例2** | 论文的Video Abstract——作者用2-3分钟视频概述研究亮点。 |
| **典型案例3** | MIT OpenCourseWare上的完整课程讲座录像。 |

#### 2.4 图组（Image Set）

| 属性 | 内容 |
|------|------|
| **精确定义** | 多张图片以版面方式组合呈现，形成整体信息单元。 |
| **属于** | 论文的多子图版面(Figure 1a-f)、长图/信息图、步骤图解、对比组合图。 |
| **不属于** | 单张图片（→静图）、网页上分散排列的独立图片、相册式浏览。 |
| **边界区分** | 与"静图"的边界：图组是多图拼合成一个版面/信息单元；静图是单张独立。判断标准：如果所有子图共用一个编号（如Figure 2）则是图组；如果各自独立编号（Figure 2a、Figure 2b分别引用）则可以是静图或图组（取决于被引用方式）。 |
| **典型案例1** | 论文中包含6张子图的完整Figure版面——Figure 1(a)到(f)合并排版。 |
| **典型案例2** | 科研方法论的长图信息图——从上到下分步骤展示实验流程，各步骤配图。 |
| **典型案例3** | 实验中"处理前vs处理后"的并排对比图组。 |

#### 2.5 交互（Interactive）

| 属性 | 内容 |
|------|------|
| **精确定义** | 用户可操控视图状态的数字化界面，含缩放、筛选、旋转、点击等交互行为。 |
| **属于** | 交互地图（如ArcGIS嵌入）、数据仪表盘、3D模型浏览器、可缩放全景图、网页端可视化工具。 |
| **不属于** | 静态截图展示仪表盘但不可操作（→静图/图组）、简单的图片幻灯片切换（→图组）。 |
| **边界区分** | 与"视频"的边界：交互是用户驱动、非线性浏览；视频是线性播放、被动观看。与"静图"的边界：交互要求用户可改变视图状态，静图固定不变。 |
| **典型案例1** | Allen Brain Atlas的交互式脑图谱网页——可切换脑区、放大缩小、点击查看细胞数据。 |
| **典型案例2** | NASA的全球气候数据3D可视化仪表盘——用户可以旋转地球、筛选年份、切换数据层。 |
| **典型案例3** | 论文学术出版商网站上嵌入的"Deep Zoom"可缩放超高分辨率显微图。 |

#### 2.6 实体（Physical）

| 属性 | 内容 |
|------|------|
| **精确定义** | 以物理实体形式存在的视觉载体，通过印刷、展示、包装等方式呈现。 |
| **属于** | 印刷海报、会议展板、宣传册、期刊纸质封面、产品包装盒上的视觉设计。 |
| **不属于** | 海报的电子设计稿（→静图/图组）、展览屏幕上的轮播视频（→视频）。 |
| **边界区分** | 判断标准：影像的最终"归宿"是否为物理实体。同一张图片，在网页上浏览是静图，被印刷成海报张贴在会议现场就是实体。 |
| **典型案例1** | 学术会议现场张贴的研究海报（Poster）——A0大幅面印刷。 |
| **典型案例2** | Nature纸质期刊的封面——经过印刷工艺呈现的实体封面。 |
| **典型案例3** | 科研设备展销会上的产品展板——易拉宝、背景板等实体印刷物。 |

### 轴三：技术手段（How-Produce）

影像是通过什么技术工具和工艺被生产出来的。

#### 3.1 拍摄（Photography）

| 属性 | 内容 |
|------|------|
| **精确定义** | 通过相机（含手机、无人机）以光学方式采集现实场景的影像。 |
| **属于** | 纪实摄影、人像摄影、航拍、延时摄影、水下摄影、微距摄影（镜头放大，非显微镜）。 |
| **不属于** | 通过显微镜成像系统采集的图像（→成像）、以摄影方式采集后经AI大幅生成的图像（→生成）。 |
| **边界区分** | 与"成像"的边界：拍摄是用常规光学镜头直接成像；成像需要借助特殊仪器系统（显微镜、MRI等）。判断标准：设备是否为通用相机/摄像机vs专用科学仪器。 |
| **典型案例1** | 科研人员在实验室被自然光/摄影灯光照射下拍摄的肖像照。 |
| **典型案例2** | 大疆无人机拍摄的野外生态研究站点全景照片。 |
| **典型案例3** | 科研纪录片中拍摄的实验操作过程影像。 |

#### 3.2 成像（Instrument Imaging）

| 属性 | 内容 |
|------|------|
| **精确定义** | 通过科学仪器系统采集的、超越人眼可见范围或分辨率的影像，核心依赖仪器而非镜头。 |
| **属于** | 光学显微、电子显微(SEM/TEM)、共聚焦、MRI、CT、X射线、遥感卫星影像、热成像、超声。 |
| **不属于** | 使用相机的微距摄影（→拍摄）、3D模拟渲染的假色显微效果图（→渲染）。 |
| **边界区分** | 与"拍摄"的边界：成像依赖科学仪器系统、通常输出原始数据需后处理。与"渲染"的边界：成像是真实物理信号的采集，渲染是计算机模拟生成。 |
| **典型案例1** | 材料科学论文中纳米颗粒的SEM（扫描电镜）图像——展示微观形貌。 |
| **典型案例2** | 医学论文中患者脑部的MRI扫描影像——展示病灶位置。 |
| **典型案例3** | 环境科学中Landsat卫星的遥感影像——展示某地区植被覆盖变化。 |

#### 3.3 绘设（Illustration & Design）

| 属性 | 内容 |
|------|------|
| **精确定义** | 以人工或数字方式绘制、设计、排版的视觉内容，依赖设计工具和创作技巧。 |
| **属于** | 科学插画、图标设计、海报版面设计、信息图排版、平面设计合成。 |
| **不属于** | 用数据工具自动生成的图表（→数据）、用3D软件建模渲染的图像（→渲染）、AI根据提示词生成的图像（→生成）。 |
| **边界区分** | 与"生成"的边界：绘设是人类主导的主动设计创作；生成是AI或算法自动产出。与"数据"的边界：绘设强调美学和设计意图；数据强调准确映射数据值到视觉元素。 |
| **典型案例1** | 科研人员手绘后用Illustrator精修的细胞结构科学插画——用于综述文章。 |
| **典型案例2** | 为学术论文设计的Graphical Abstract——用插画和排版概括研究核心发现。 |
| **典型案例3** | 大型科研项目的LOGO和视觉识别系统设计。 |

#### 3.4 数据（Data Visualization）

| 属性 | 内容 |
|------|------|
| **精确定义** | 以数据驱动方式将结构化信息映射为图形元素（位置、形状、颜色），通过工具自动或半自动生成。 |
| **属于** | 统计图表（柱状/折线/散点/热图）、地图、网络图、桑基图、雷达图、科学图谱（如XRD/FTIR图谱）。 |
| **不属于** | 手绘/设计的非数据驱动图示（→绘设）、3D渲染生成的数据可视化（→渲染）、AI从数据生成的可视化（→生成）。 |
| **边界区分** | 与"绘设"的边界：数据可视化由数据驱动、工具自动生成图形映射；绘设由设计师手动创作。与"渲染"的边界：二维数据图表是数据可视化；三维空间中的数据呈现（如3D晶体结构）是渲染。 |
| **典型案例1** | 用Python Matplotlib生成的实验数据柱状图——高度精确对应数据值。 |
| **典型案例2** | 用Gephi生成的科研合作网络图——节点大小和连线粗细对应合作频次。 |
| **典型案例3** | 用ArcGIS/QGIS生成的科研站点地理分布地图。 |

#### 3.5 渲染（3D Rendering）

| 属性 | 内容 |
|------|------|
| **精确定义** | 基于三维数字模型，通过计算机模拟光照、材质和视角生成的二维影像。 |
| **属于** | 产品3D渲染、建筑可视化、分子结构渲染、3D数据可视化、3D人物建模渲染。 |
| **不属于** | 3D软件截图/界面导出但未经渲染计算（→绘设）、AI从文本生成3D图像（→生成）、计算机视觉的3D重建（→成像）。 |
| **边界区分** | 与"数据"的边界：渲染强调光照、材质、空间感的模拟计算。与"生成"的边界：渲染基于人工构建的3D模型和设定参数；生成基于AI模型产出。 |
| **典型案例1** | Nature封面上艺术化3D渲染的蛋白质结构——Blender/ChimeraX渲染，有光影和材质。 |
| **典型案例2** | 工程论文中的3D设备结构渲染图——展示内部组件装配关系。 |
| **典型案例3** | 建筑设计竞赛中科学实验室的空间3D效果图。 |

#### 3.6 生成（Generative）

| 属性 | 内容 |
|------|------|
| **精确定义** | 通过AI模型或算法以非确定性方式生成或显著改变的影像。 |
| **属于** | AI文本生成图像（DALL-E/Midjourney/Stable Diffusion）、算法艺术生成、AI图像增强/超分辨率、风格迁移、计算摄影的AI合成。 |
| **不属于** | 用传统滤镜/调色（→拍摄或绘设的后处理）、基于规则的程序化生成没有AI参与的（→数据/渲染）、AI辅助自动化但最终由人决定的常规设计（→绘设）。 |
| **边界区分** | 关键判断：生成过程中是否存在AI模型的非确定性推理步骤。如果只是Photoshop中的自动调色（确定性算法），不是生成；如果使用AI神经网络进行风格迁移，是生成。 |
| **典型案例1** | 用Midjourney根据蛋白质功能描述生成的科普概念图。 |
| **典型案例2** | 用AI超分辨率技术从低分辨率显微图生成的高清版本——显著改变了像素信息。 |
| **典型案例3** | 用DeepDream/Neural Style Transfer将科学图像转化为特定艺术风格的实验作品。 |

## 第二部分：三轴×6维映射表

现有6维分类体系：
- `mediaType`: 摄影、手绘图、3D渲染、信息图、显微图、数据可视化、期刊封面、混合媒介、视频、PDF/文档
- `contentType`: 科研人员、实验设备、实验过程、微观样本、机制模型、数据结果、空间环境、团队场景、科普传播
- `discipline`: 生命科学、材料、医学、工程、物理、化学、信息科学、环境科学、综合交叉
- `visualStyle`: 纪实、科技、极简、人文、艺术化、教学解释、商业宣传、顶刊封面
- `composition`: AI分析产出
- `colorTone`: AI分析产出

### 2.1 功能用途 ← 现有维度映射

功能用途主要从 `contentType` + `visualStyle` + `useCase` 三个现有字段综合推断。

| 功能用途 | 推断规则（优先级从高到低） | 关键字段 | 说明 |
|----------|--------------------------|---------|------|
| **记录** | (1) contentType=科研人员 AND visualStyle=纪实 | contentType, visualStyle | 人物纪实照片 |
| | (2) contentType=实验设备 AND visualStyle=纪实 | contentType, visualStyle | 设备现场记录 |
| | (3) contentType=实验过程 AND visualStyle=纪实 | contentType, visualStyle | 过程记录 |
| | (4) contentType=空间环境 AND visualStyle=纪实 | contentType, visualStyle | 环境记录 |
| | (5) contentType=团队场景 | contentType | 团队合影记录 |
| | (6) useCase 包含"记录/存档/纪实/现场" | useCase | 语义匹配兜底 |
| **解释** | (1) contentType=机制模型 AND visualStyle=教学解释 | contentType, visualStyle | 明确的教学解释意图 |
| | (2) contentType=机制模型 AND visualStyle≠商业宣传 | contentType, visualStyle | 机制模型通常用于解释 |
| | (3) mediaType=信息图 AND visualStyle=教学解释 | mediaType, visualStyle | 信息图的解释功能 |
| | (4) visualStyle=教学解释 AND contentType≠实验过程/团队场景 | visualStyle, contentType | 教学解释风格的解释意图 |
| | (5) useCase 包含"解释/说明/机制/原理/流程" | useCase | 语义匹配兜底 |
| **数据** | (1) contentType=数据结果 | contentType | 最直接的映射 |
| | (2) mediaType=数据可视化 | mediaType | 数据可视化呈现数据 |
| | (3) mediaType=显微图 AND visualStyle=科技 | mediaType, visualStyle | 科技风格的仪器数据 |
| | (4) useCase 包含"数据/统计/结果/趋势/实验" | useCase | 语义匹配兜底 |
| **展示** | (1) contentType=科研成果 AND visualStyle=科技 | contentType, visualStyle | 成果展示 |
| | (2) contentType=空间环境 AND visualStyle≠纪实 | contentType, visualStyle | 美化后的空间展示 |
| | (3) mediaType=3D渲染 AND visualStyle=科技 | mediaType, visualStyle | 3D产品/成果展示 |
| | (4) visualStyle=极简 AND contentType≠机制模型 | visualStyle, contentType | 极简风格的展示化呈现 |
| | (5) useCase 包含"展示/成果/产品/项目/形象" | useCase | 语义匹配兜底 |
| **传播** | (1) visualStyle=顶刊封面 | visualStyle | 期刊封面直接对应传播 |
| | (2) visualStyle=商业宣传 | visualStyle | 商业宣传直接对应传播 |
| | (3) mediaType=期刊封面 | mediaType | 期刊封面即传播 |
| | (4) contentType=科普传播 | contentType | 科普即传播 |
| | (5) visualStyle=艺术化 AND contentType≠机制模型 | visualStyle, contentType | 艺术化用于传播 |
| | (6) useCase 包含"传播/科普/品牌/封面/宣传/公众" | useCase | 语义匹配兜底 |
| **交互** | (1) mediaType=混合媒介 AND 来源域包含交互关键词 | mediaType, sourceDomain | 混合媒介中可能含交互 |
| | (2) useCase 包含"交互/浏览/筛选/探索/仪表盘/可缩放" | useCase | 主要靠语义匹配（现有字段对此覆盖不足） |
| | (3) contentType=数据结果 AND 来源域=可视化平台 | contentType, sourceDomain | 仪表盘类交互 |

> **注意**：`useCase`字段目前尚未系统化启用，是未来自动化推断的主要支撑字段。在上述规则中，`useCase`规则作为兜底规则，优先级最低。随着`useCase`字段的充实，可逐步调整优先级权重。

### 2.2 传播媒介 ← 现有维度映射

`distributionMedium` 是**完全新增的维度**，当前数据库不直接覆盖，需从 `mediaType`、`sourceDomain`、`captureType` 等字段综合推断。

| 传播媒介 | 推断规则（优先级从高到低） | 关键字段 | 说明 |
|----------|--------------------------|---------|------|
| **静图** | (1) mediaType IN (摄影, 手绘图, 3D渲染, 显微图) AND captureType=image | mediaType, captureType | 最常见的组合 |
| | (2) mediaType=数据可视化 AND sourceDomain NOT IN (仪表盘平台域名) | mediaType, sourceDomain | 静态数据图 |
| | (3) mediaType=信息图 | mediaType | 信息图通常为静图 |
| | (4) captureType=image AND mediaType≠视频 | captureType, mediaType | 兜底规则 |
| **动图** | (1) captureType=image AND imageUrl 后缀为 .gif | captureType, imageUrl | 文件格式判断 |
| | (2) mediaType=视频 AND captureType=image | mediaType, captureType | 被当图片捕获的动态内容 |
| | (3) sourceDomain 含 giphy/tenor 等动图平台 | sourceDomain | 来源特征 |
| **视频** | (1) mediaType=视频 | mediaType | 直接映射 |
| | (2) captureType=video | captureType | 捕获类型即视频 |
| | (3) sourceDomain IN (youtube.com, bilibili.com, vimeo.com) | sourceDomain | 视频平台来源 |
| **图组** | (1) captureType=image AND sourceUrl 含 figure/multi-panel | captureType, sourceUrl | URL特征 |
| | (2) mediaType=信息图 AND contentText 含"Figure 1(a-f)"等多子图描述 | mediaType, contextText | 多图版面 |
| | (3) mediaType=PDF/文档 | mediaType | 文档常含多图版面 |
| | (4) 同一页面同一编号下有多个子图标注 | contextText | 文本特征推断 |
| **交互** | (1) sourceDomain IN (可视化平台域名列表) | sourceDomain | 平台特征最直接 |
| | (2) caption 含"interactive/zoom/explore/dashboard/3D viewer" | contextText | 文本语义特征 |
| | (3) mediaType=混合媒介 AND sourceDomain≠静态页面 | mediaType, sourceDomain | 混合媒介可能含交互 |
| | (4) useCase包含"交互/浏览/探索/仪表盘" | useCase | 语义匹配 |
| **实体** | (1) sourceDomain 含 poster/conference/print/gallery | sourceDomain | 来源推测 |
| | (2) caption含"海报/poster/展板/印刷/封面/包装" | contextText | 文本特征 |
| | (3) mediaType=期刊封面 AND visualStyle=顶刊封面 | mediaType, visualStyle | 纸质封面 |
| | (4) sourceType=exhibition/print | sourceType | CrawlSource表的来源类型 |

> **新增字段建议**：`distributionMedium` (String, 枚举值: static_image/animated_image/video/image_set/interactive/physical)，默认值 `static_image`。人工标注优先级最高，上述规则用于自动推断兜底。

### 2.3 技术手段 ← mediaType 映射

技术手段与 `mediaType` 的映射关系最为直接，同时也参考 `contentType` 和 `visualStyle` 做细化。

| 技术手段 | mediaType 直接映射 | 辅助推断（contentType + visualStyle） | 冲突处理 |
|----------|-------------------|--------------------------------------|---------|
| **拍摄** | 摄影 | contentType=科研人员/实验设备/实验过程/团队场景/空间环境；visualStyle=纪实/人文 | 如同时匹配"摄影"和其他mediaType，以mediaType为准 |
| **成像** | 显微图 | contentType=微观样本；visualStyle=科技 | — |
| **绘设** | 手绘图、信息图、期刊封面 | visualStyle=艺术化/教学解释 | 期刊封面通常为绘设+渲染混合，按主要技术手段判断 |
| **数据** | 数据可视化 | contentType=数据结果 | — |
| **渲染** | 3D渲染 | contentType=机制模型/微观样本；visualStyle=科技/顶刊封面 | — |
| **生成** | —（当前mediaType无此值） | 需要新增 mediaType=AI生成 或在useCase中标注 | 当前库中暂无此类别案例，为未来预留 |

#### 技术手段推断决策树

```
mediaType=摄影       → 技术手段=拍摄
mediaType=显微图     → 技术手段=成像
mediaType=手绘图     → 技术手段=绘设
mediaType=信息图     → 技术手段=绘设
mediaType=3D渲染     → 技术手段=渲染
mediaType=数据可视化  → 技术手段=数据
mediaType=期刊封面   → 技术手段=绘设（默认）或渲染（若contentType=机制模型）
mediaType=视频       → 技术手段=拍摄（默认）或绘设（若contentType=机制模型）+ 新增动效标注
mediaType=混合媒介   → 技术手段需要人工判断，目前无法自动推断
mediaType=PDF/文档   → 技术手段需要人工判断，目前无法自动推断
```

## 第三部分：子分类字典

子分类是对三轴主类的进一步细分，用于更精细的编码和分析。编码采用 `主类编码-子类编码` 格式（如 `REN-PRODUCT`）。

### 3.1 技术手段子分类

#### 3.1.1 渲染（REN）子分类

| 编码 | 名称 | 定义 | 典型案例 |
|------|------|------|---------|
| `REN-PRODUCT` | 3D产品渲染 | 对产品外观、材质、光影进行的三维渲染展示，强调质感真实感，面向展示或传播。 | Apple官网上iPhone的3D产品渲染图；追觅扫地机器人的材质精修渲染。 |
| `REN-MECHANISM` | 3D机制图 | 用3D建模和渲染呈现科学/工程机制的结构与原理，通常含标注和透明度展示内部结构。 | Nature封面上的3D蛋白结构渲染图；Cell中展示病毒入侵机制的3D渲染。 |
| `REN-SCENE` | 3D场景渲染 | 对空间、环境、建筑、地貌的三维场景渲染，用于展示空间关系或模拟真实环境。 | 科研建筑竞赛的效果图；模拟火星表面探测器工作的场景渲染。 |
| `REN-DATAVIZ` | 3D数据可视化 | 将数据映射到三维空间的渲染影像，如3D流场、晶体结构、分子模拟等。 | 流体力学论文中的3D湍流结构渲染图；蛋白质数据库的3D分子展示。 |
| `REN-PORTRAIT` | 3D人物/肖像 | 以三维建模和渲染生成的人物或群体肖像，通常用于科普动画或虚拟场景。 | 科普视频中科学家的3D虚拟形象；学术会议虚拟展厅中的数字人导览。 |

#### 3.1.2 拍摄（PHO）子分类

| 编码 | 名称 | 定义 | 典型案例 |
|------|------|------|---------|
| `PHO-DOCUMENTARY` | 纪实摄影 | 以客观记录为首要目的的拍摄，不干预场景、不刻意美化，强调真实性和现场感。 | MIT实验室网页上研究人员自然操作的抓拍；CERN加速器隧道的现场照片。 |
| `PHO-PORTRAIT` | 人像摄影 | 以人物为主要拍摄对象的摄影，细分为个人肖像和群体肖像。 | 科研团队网站上的PI个人肖像照；实验室年终合影。 |
| `PHO-PROCESS` | 过程摄影 | 以记录实验过程、操作步骤、时序变化为核心的摄影，通常为系列图片。 | 论文补充材料中展示样品制备步骤的过程摄影；材料腐蚀变化的时间序列拍摄。 |
| `PHO-AERIAL` | 航拍 | 通过无人机或其他飞行器从空中拍摄的影像，提供宏观视角。 | 国家野外科学观测研究站的全景航拍图；大型科学设施（如FAST射电望远镜）的俯瞰图。 |

> **PHO-PORTRAIT 细分（可按需开启）**：
> - `PHO-PORTRAIT-INDIVIDUAL`: 单个人物肖像
> - `PHO-PORTRAIT-GROUP`: 群体/团队肖像

#### 3.1.3 绘设（DES）子分类

| 编码 | 名称 | 定义 | 典型案例 |
|------|------|------|---------|
| `DES-ILLUSTRATION` | 科学插画 | 手绘或数字方式创作的科学概念插画，强调对科学内容的艺术化表达和概念传达。 | Nature Reviews中的综述插画；教科书中用数位板绘制的人体解剖图。 |
| `DES-ICON` | 图标设计 | 为科研内容设计的图标、符号、LOGO等图形元素，强调识别性和符号化表达。 | 科研项目中用AI设计的信号通路图标；实验室LOGO和视觉符号。 |
| `DES-LAYOUT` | 版面设计/信息图 | 以排版、布局和信息层次设计为核心，将多元素组合成统一视觉版面。 | 论文的Graphical Abstract（含文字+插画+图标+数据小图）；学术会议的海报设计。 |

#### 3.1.4 成像（IMG）子分类

| 编码 | 名称 | 定义 | 典型案例 |
|------|------|------|---------|
| `IMG-LIGHT` | 光学显微 | 通过光学显微镜采集的可见光影像，包括明场、相差、荧光、共聚焦等模式。 | 生物学论文中细胞骨架的荧光显微照片；病理学中的H&E染色组织切片。 |
| `IMG-ELECTRON` | 电子显微 | 通过电子显微镜（SEM/TEM/STEM）采集的影像，可达到纳米级分辨率。 | 材料科学论文中纳米颗粒形貌的SEM图像；病毒颗粒的冷冻电镜(Cryo-EM)结构。 |
| `IMG-MEDICAL` | 医学影像 | 通过医学成像设备采集的用于临床诊断或研究的影像，包括MRI、CT、PET、超声、X射线。 | 脑科学研究中fMRI脑区激活图；放射学论文中的肺部CT影像。 |
| `IMG-REMOTE` | 遥感成像 | 通过卫星、航空器遥感传感器采集的地球表面影像，通常覆盖大面积区域。 | Landsat卫星的森林覆盖变化监测图；MODIS数据的海洋叶绿素浓度分布图。 |
| `IMG-THERMAL` | 热成像 | 通过红外热成像仪采集的表面温度分布影像，呈现不可见的热辐射信息。 | 工程论文中电子器件散热分布的热成像图；生态学中动物体温监测的红外热图。 |

#### 3.1.5 数据（DAT）子分类

| 编码 | 名称 | 定义 | 典型案例 |
|------|------|------|---------|
| `DAT-STATISTICAL` | 统计图表 | 以统计量（均值、标准差、P值等）为核心的图表，包括柱状图、散点图、箱线图、热图等。 | 临床研究论文中的三组疗效比较柱状图（含误差线和显著性标记）。 |
| `DAT-MAP` | 地图 | 以地理空间数据为核心的二维可视化，用颜色、符号、大小编码空间属性。 | 流行病学研究中用ArcGIS制作的新冠病例地理分布图。 |
| `DAT-NETWORK` | 网络图 | 以节点和边呈现关系数据的可视化，包括社交网络、合作网络、知识图谱、生物网络。 | 用Cytoscape绘制的蛋白质相互作用(PPI)网络图；科研合作者网络。 |
| `DAT-SCIENTIFIC` | 科学数据可视化 | 针对科学仪器或模型输出的专业数据类型进行可视化，如光谱图、色谱图、XRD图谱、相图、能带图。 | 化学论文中的核磁共振(NMR)氢谱；半导体论文中的能带结构图。 |

#### 3.1.6 生成（GEN）子分类

| 编码 | 名称 | 定义 | 典型案例 |
|------|------|------|---------|
| `GEN-AI-IMAGE` | AI图像生成 | 通过文本到图像、图像到图像等AI模型从头生成或大幅创作的全新图像。 | 用DALL-E根据蛋白质功能描述生成的科普概念图；Midjourney生成的科学场景概念图。 |
| `GEN-ALGORITHMIC` | 算法生成 | 通过确定性算法（非AI）程序化生成的艺术或科学图像，如分形、L-System、反应扩散等。 | 用Mandelbrot集生成的分形图作为科学演讲背景；用L-System模拟植物生长的算法图像。 |
| `GEN-ENHANCEMENT` | 图像增强 | 通过AI对已有图像进行超分辨率、去噪、去模糊等质量增强处理。 | 用AI超分辨率从128×128低分显微图生成的1024×1024高清版本。 |
| `GEN-STYLE` | 风格迁移 | 通过AI将已有图像转换为另一种视觉风格，保留内容结构的同时改变纹理和色彩风格。 | 用Neural Style Transfer将普通显微图转换为梵高风格的艺术科学图像。 |

### 3.2 功能用途子分类

#### 3.2.1 记录（REC）子分类

| 编码 | 名称 | 定义 | 典型案例 |
|------|------|------|---------|
| `REC-PORTRAIT` | 个人肖像 | 记录单个科研人员的形象，用于官网个人页、学术主页、简介配图。 | 大学官网上教授的标准肖像照；Google Scholar学术主页头像。 |
| `REC-TEAM` | 群体/团队 | 记录科研团队或群体的合影、工作场景，用于团队介绍、年度报告。 | 实验室主页上的全体成员合影；学术会议后的参会者合照。 |
| `REC-PROCESS` | 实验过程 | 以存档或记录为目的的实验操作、步骤、流程影像。 | 实验记录本中附带的操作步骤照片；补充材料中展示样本制备过程。 |
| `REC-EQUIPMENT` | 设备空间 | 记录实验设备、仪器、实验室/车间空间状态，用于设备展示或空间存档。 | 官网实验室介绍页中的设备照片；设备使用手册中的装置外观记录。 |
| `REC-ENVIRONMENT` | 现场环境 | 记录科研工作所处的自然或人工环境全貌，强调场景整体感。 | 野外考察站点的全景照片；医院临床研究基地的环境记录。 |

#### 3.2.2 解释（EXP）子分类

| 编码 | 名称 | 定义 | 典型案例 |
|------|------|------|---------|
| `EXP-MECHANISM` | 机制图解 | 解释生物/化学/物理/工程机制的视觉图解，通常展示因果链和相互作用。 | Cell论文中的信号通路图——展示配体→受体→级联反应→转录。 |
| `EXP-STRUCTURE` | 结构剖析 | 解释对象（分子、细胞、器件、建筑物）内部结构的视觉图解，通常含剖面或分解图。 | 工科论文中的设备结构剖面图——标注各部件名称与功能。 |
| `EXP-PROCESS` | 流程步骤 | 解释从A到B的过程、步骤或时序的视觉图解，通常含箭头和编号。 | 综述论文中的实验方法流程图——从样品制备到数据分析的完整步骤。 |
| `EXP-PRINCIPLE` | 原理示意 | 解释抽象科学原理或理论的视觉示意，通常侧重概念层面的理解。 | 物理学论文中的相对论时空弯曲示意图；教科书中的DNA半保留复制原理图。 |

#### 3.2.3 功能-数据（DAT）子分类

| 编码 | 名称 | 定义 | 典型案例 |
|------|------|------|---------|
| `DAT-RESULT` | 实验结果 | 呈现实验或实证研究的具体结果数据，强调对假设的验证或拒绝。 | 新药临床试验论文中的疗效对比柱状图（含P值）。 |
| `DAT-STATISTICS` | 统计分析 | 呈现统计分析过程或结果，包括回归、聚类、显著性检验等。 | 基因关联分析(GWAS)的曼哈顿图(Manhattan plot)。 |
| `DAT-MODEL` | 模型模拟 | 呈现计算模型或数值模拟的输出结果，如预测曲线、模拟快照。 | 气候模型中未来50年温度变化的预测曲线；分子动力学模拟的轨迹分析图。 |
| `DAT-TREND` | 趋势预测 | 呈现时间序列数据的趋势、周期和预测，强调变化方向。 | 文献计量学分析中的发文量年度趋势图；流行病学中疫情传播的预测曲线。 |

#### 3.2.4 展示（SHO）子分类

| 编码 | 名称 | 定义 | 典型案例 |
|------|------|------|---------|
| `SHO-PRODUCT` | 产品展示 | 以展示科研相关产品外观、功能、质感为核心的影像。 | 测序仪公司的测序仪产品3D渲染图；实验耗材公司的产品目录图。 |
| `SHO-PROJECT` | 项目形象 | 以展示科研项目整体形象和核心亮点为目的的视觉内容。 | 国家重点研发计划的项目展示海报；基金申请书中的项目概览图。 |
| `SHO-SPACE` | 空间展示 | 以展示科研空间（实验室、创新中心、科技园区）的设计和品质为核心的影像。 | 新建实验室的效果图；科学城规划的建筑可视化渲染。 |
| `SHO-ACHIEVEMENT` | 成果展示 | 以展示科研成果（获奖、专利、论文封面、媒体报道）为核心的影像。 | 获得国家科学技术奖时配套的成果展示图；研究院年报中的成果亮点页面。 |

#### 3.2.5 传播（OUT）子分类

| 编码 | 名称 | 定义 | 典型案例 |
|------|------|------|---------|
| `OUT-BRAND` | 品牌宣传 | 以塑造机构/课题组/科研平台的品牌形象为目的的视觉内容。 | 大学官网首页轮播图中的科研场景横幅；研究院的官方宣传片截图。 |
| `OUT-SCIENCE` | 科普传播 | 以向公众普及科学知识为目的的视觉内容，强调易懂性和趣味性。 | 果壳网/Sheldon科学漫画中的科普图解；B站科普UP主的视频封面。 |
| `OUT-COVER` | 封面 | 专为期刊、书籍封面设计的视觉内容，强调艺术性和代表性。 | Nature当期封面；科学出版社专著的封面设计。 |
| `OUT-RECRUITMENT` | 招生 | 以吸引学生/研究人员申请为目的的视觉内容。 | 暑期学校招生海报；课题组招募博士后的社交媒体配图。 |
| `OUT-GRANT` | 项目申报 | 为科研项目申报（基金申请、项目答辩）配套的视觉内容。 | 国家自然科学基金申请书中的技术路线图；答辩PPT中的项目架构图。 |

#### 3.2.6 交互（INT）子分类

| 编码 | 名称 | 定义 | 典型案例 |
|------|------|------|---------|
| `INT-EXPLORE` | 数据探索 | 支持用户在多维数据中自由探索、下钻、筛选、排序的交互界面。 | Gapminder的动态气泡图工具——用户可切换国家、年份和指标。 |
| `INT-FILTER` | 交互筛选 | 支持用户通过条件筛选缩小或切换数据范围的交互界面。 | COVID-19数据仪表盘中的国家/时间筛选器；相亲图(phase diagram)的在线交互筛选器。 |
| `INT-VISUAL` | 可视化浏览 | 支持用户在视觉空间中自由浏览、缩放、旋转的交互工具。 | Allen Brain Atlas的脑图谱交互浏览器；可缩放超高分辨率全景组织切片浏览器。 |
| `INT-DECISION` | 决策支持 | 为科研或管理决策提供交互式分析支持的仪表盘界面。 | 科研项目管理系统中的多项目进度与产出仪表盘；实验室资源配置决策面板。 |

| `INTER-DASHBOARD` | 数据仪表盘 | 以多图表整合和实时更新为特征的交互数据面板。 | Johns Hopkins大学COVID-19全球追踪仪表盘；研究机构内部的项目统计仪表盘。 |
| `INTER-3DVIEWER` | 3D浏览器 | 支持用户自由旋转、缩放、剖切三维模型的交互查看器。 | 蛋白质数据库(PDB)的Mol* 3D分子浏览器；医学教育中的3D解剖模型浏览器。 |
| `INTER-VISUALIZATION` | 网页可视化 | 嵌入网页中、支持用户交互（悬停、点击、筛选）的定制数据可视化。 | New York Times新闻报道中嵌入的交互式数据图表；Nature论文在线版的交互式图表。 |
