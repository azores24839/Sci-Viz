## C1. 企业视觉案例采集

### 目标

确定对标企业清单，爬取企业视觉素材，入库 Sci-Viz Case Hub，为"科研视觉商业化路径分析"提供数据基础。

### 对标企业清单（初步）

| 企业 | 为什么对标 | 视觉特点 |
|------|----------|---------|
| 追觅 | 中国科技企业、产品3D渲染标杆 | 产品外观3D渲染极致 |
| 小米 | 中国科技品牌、生态链视觉体系 | 产品渲染+生活场景融合 |
| 苹果 | 全球产品视觉标杆 | 极简+材质细节+动效 |
| Tesla | 科技产品如何做"硬核技术视觉化" | 技术动画+数据可视化+产品渲染 |
| Boston Dynamics | 机器人技术如何视觉呈现 | 技术+运动+场景纪实 |
| SpaceX | 最强科技品牌视觉叙事 | 发射纪实+工程图+数据可视化 |

### 具体验收标准

1. **企业来源列表**：6-8个企业，每个标注视觉价值、采集难度、采集策略
2. **爬虫适配器**：为每个企业编写静态来源适配器（官网产品页、新闻页、技术博客）
3. **入库数量**：每个企业 15-30 条高质量案例，总计 100-200 条
4. **source_type 标记**：所有企业案例标记为 `enterprise`
5. **三轴标注**：企业案例需有基础的 mediaType、functionalPurpose 标注
6. **低价值图片 < 20%**

### 可能遇到的挑战

| 挑战 | 应对策略 |
|------|---------|
| 企业官网反爬（Cloudflare等） | 优先走静态HTML适配器；受阻则手动采集+浏览器插件 |
| 产品图片多为纯白背景图 | 这是企业案例的特征，保留但标注functionalPurpose=展示 |
| 企业来源视频多 | 视频→distributionMedium=视频，functionalPurpose=传播 |

### 产出

- CRAWL_SOURCE_TARGETS.md 更新（新增企业来源段落）
- 爬虫适配器代码
- 入库企业案例统计报告

### 估算工作量

- 来源清单和适配器编写：3-4小时/企业
- 爬取和入库：自动化
- 质量检查：1-2小时
- 合计：20-30小时（含6-8个企业）
