## A1. 三轴分类体系 + 映射表 + 子分类字典

### 目标

完成三轴分类体系的最终定义，建立三轴与现有6维的映射关系，完成所有主类的子分类字典。

### 具体验收标准

1. **三轴选项字典定稿**：功能用途6类、传播媒介6类、技术手段6类，每类有清晰定义和边界说明
2. **映射表完成**：三轴的每个选项与现有6维（mediaType/contentType/discipline/visualStyle/composition/colorTone）的对应关系明确，能回答"一张已有案例的三轴标签是什么"
3. **子分类字典完成**：至少覆盖以下主类的子分类：
   - 技术→渲染：3D产品渲染、3D机制图、3D场景渲染、3D数据可视化、3D人物/肖像
   - 技术→拍摄：纪实摄影、人像摄影（个人/群体）、过程摄影、航拍
   - 技术→绘设：科学插画、图标设计、版面设计
   - 功能→记录：个人肖像、群体/团队、实验过程、设备空间
   - 其他主类视数据覆盖情况补充
4. **人审定完成**：每项定义和边界经过人工确认

### 产出文件

- 三轴选项字典文档（含定义和边界说明）
- 三轴×6维映射表（可执行为数据库映射或文档表格）
- 子分类字典文档

### 人机分工

| 任务 | AI做 | 人做 |
|------|------|------|
| 起草选项字典和映射表 | 初稿 | 审定边界、确认歧义 |
| 起草子分类字典 | 初稿 | 确认分类是否完整 |
| 映射表测试（对现有案例打标） | 批量映射 | 抽检20-30个案例 |

## A2. 数据库与系统改造

### 目标

在 Sci-Viz Case Hub 中新增三轴维度的数据字段、标注界面和分析功能。

### 具体验收标准

1. **数据库 schema 变更**：新增4个字段
   - `functionalPurpose`（功能用途）：枚举，映射自 contentType + useCase
   - `distributionMedium`（传播媒介）：枚举，新增维度
   - `mediaSubType`（技术手段子分类）：枚举，映射自 mediaType 细分
   - `contentSubType`（内容对象子分类）：枚举，映射自 contentType 细分
2. **前端标注界面**：案例详情页和复核页能标注三轴标签和子分类
3. **Insights 页面**：支持按三轴维度筛选、分布图表、交叉矩阵
4. **批量映射脚本**：现有2837+案例自动映射到三轴标签（基于现有6维字段）
5. **映射覆盖率 > 85%**：自动映射后，85%以上的案例有完整三轴标签

### 产出

- Prisma migration 文件
- 后端 API 变更
- 前端标注和洞察页面更新
- 批量映射脚本及其执行结果报告

### 人机分工

|------|------|------|
| 数据库 schema 设计和 migration | 完全做 | 确认字段和枚举值 |
| 后端 API 扩展 | 完全做 | 确认接口语义 |
| 前端标注/分析页面 | 完全做 | 确认交互逻辑 |
