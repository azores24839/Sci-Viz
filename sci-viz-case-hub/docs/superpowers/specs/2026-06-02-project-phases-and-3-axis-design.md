# 科研影像项目阶段规划与三轴分类体系设计

> 日期：2026-06-02
> 状态：设计确认，待实施

---

## 一、项目总目标

**制定科研影像创作流程——商业性**：为科研公司/团队提供专业科研影像服务，从"各领域呈现公式"出发，提升科研成果的展示质量。

当前痛点：很多科研成果呈现质量差，但我们能整理出各领域的呈现规律和最佳实践，形成可复用的方法论。

---

## 二、三轴分类体系

三轴分类法是贯穿整个项目的**上层分析框架**，回答三个独立问题：

| 轴 | 问题 | 选项 |
|---|---|---|
| **功能用途** | 影像为何存在？ | 记录、解释、数据、展示、传播、交互 |
| **传播媒介** | 影像如何被观看与传播？ | 静图、动图、视频、图组、交互、实体 |
| **技术手段** | 影像如何被生产？ | 拍摄、成像、绘设、数据、渲染、生成 |

### 2.1 功能用途

| 类别 | 含义 |
|---|---|
| 记录 | 记录对象、现场、人物、实验过程 |
| 解释 | 说明机制、结构、流程、原理 |
| 数据 | 呈现实验结果、统计、模型、趋势 |
| 展示 | 展示成果、产品、空间、项目形象 |
| 传播 | 面向公众、品牌、科普、封面、宣传 |
| 交互 | 支持操作、浏览、筛选、探索、决策 |

### 2.2 传播媒介

| 类别 | 含义 |
|---|---|
| 静图 | JPG、PNG、TIFF、SVG、网页图片 |
| 动图 | GIF、动态图、循环动效 |
| 视频 | 短视频、动画、纪录片、讲解视频 |
| 图组 | 多图版面、论文组图、长图、步骤图 |
| 交互 | 交互地图、仪表盘、可缩放图像、网页可视化 |
| 实体 | 印刷物、海报、展板、展览屏幕、包装 |

### 2.3 技术手段

| 类别 | 含义 |
|---|---|
| 拍摄 | 摄影、摄像、航拍、现场采集 |
| 成像 | 显微、医学、遥感、热成像、仪器图像 |
| 绘设 | 插画、图标、平面设计、排版、合成 |
| 数据 | 图表、地图、网络图、数据可视化 |
| 渲染 | 3D、仿真、建模、分子/工程渲染 |
| 生成 | AI 生成、算法生成、图像增强、风格迁移 |

### 2.4 三轴与现有6维的关系

三轴是上层分析框架，现有6维是底层数据标签。两者共存，通过映射表连接：

| 三轴维度 | 映射自现有维度 |
|---|---|
| 功能用途 | ← contentType + useCase（useCase字段已有但尚未启用） |
| 传播媒介 | ← 新增字段（当前数据库缺失此维度） |
| 技术手段 | ← mediaType（含子分类） |

映射关系示例：
- 一张"Nature封面的3D蛋白渲染图"：功能=传播+解释，媒介=静图，技术=渲染
- 一段"MIT实验室视频"：功能=记录+展示，媒介=视频，技术=拍摄

### 2.5 子分类体系

三轴的每个类别都可以进一步细分（子分类），按用途和内容对象展开：

#### 技术手段→渲染 子分类：

| 子类 | 说明 |
|---|---|
| 3D产品渲染 | 产品外观、材质、光影展示 |
| 3D机制图 | 分子、细胞、工程结构原理 |
| 3D场景渲染 | 空间、环境、建筑、地貌 |
| 3D数据可视化 | 三维数据、流场、晶体结构 |
| 3D人物/肖像 | 人物肖像、群体人物建模 |

#### 技术手段→拍摄 子分类：

| 子类 | 说明 |
|---|---|
| 纪实摄影 | 实验室、现场、设备、环境 |
| 人像摄影 | 个人肖像、群体肖像、团队合影 |
| 过程摄影 | 实验过程、操作步骤 |
| 航拍 | 无人机拍摄、大场景 |

#### 技术手段→绘设 子分类：

| 子类 | 说明 |
|---|---|
| 科学插画 | 手绘/数字插画、概念图 |
| 图标设计 | UI图标、科学符号 |
| 版面设计 | 版面、海报、信息图排版 |

#### 功能用途→记录 的子分类（内容对象维度）：

| 子类 | 说明 |
|---|---|
| 个人肖像 | 单人科研人员 |
| 群体/团队 | 多人合影、团队场景 |
| 实验过程 | 操作、步骤、过程记录 |
| 设备空间 | 仪器、实验室、空间 |

子分类在数据库中通过 `mediaSubType` 和 `contentSubType` 字段实现，与三轴大类一对多映射。

## 三、阶段规划

### 阶段A：概念地基（与阶段B可并行）

#### A1. 三轴分类体系 + 映射表 + 子分类字典

**产出：**
- 三轴选项字典最终版（2.1-2.3已完成）
- 三轴×6维映射表
- 各主类的通用子分类字典
- 数据库字段变更方案（新增 `mediaSubType`、`contentSubType`、`distributionMedium`、`functionalPurpose`）

**人机分工：**
- AI：起草映射表初稿、子分类字典初稿
- 人：审定每个映射关系、确认子分类边界

#### A2. 数据库与系统改造

**产出：**
- 数据库 schema 新增4个字段：`functionalPurpose`、`distributionMedium`、`mediaSubType`、`contentSubType`
- 系统中新增三轴筛选/标注界面
- Insights 页面支持三轴维度分布和交叉矩阵
- 编辑案例时可标注三轴和子分类

---

### 阶段B：交大深研（与阶段A可并行）

#### B1. 交大学科→顶刊映射

**产出：**
- 上海交大学科列表（对应学院/系）
- 每个学科对应的国内顶刊名单
- 每个学科对应的国际顶刊名单

**人机分工：**
- AI：检索交大所有学院/系信息，整理学科列表
- AI：检索各学科国内外顶刊目录
- 人：确认学科边界和顶刊选择标准

#### B2. 交大二级页面深度爬取

**产出：**
- 交大各学院/实验室/团队的二级网站URL列表
- 爬取这些页面的视觉内容（实验室照片、团队合影、设备照片、成果配图）
- 入库 Sci-Viz Case Hub

**人机分工：**
- AI：自动发现交大二级页面链接（从主站页面提取学院/实验室链接）
- AI：编写爬虫适配器，爬取二级页面
- 人：确认哪些实验室/团队有价值、检验爬取质量

#### B3. 交大各学科呈现方式比例统计

**产出：**
- 交大各学科网页的呈现方式分布：静图占比、视频占比、3D渲染占比、信息图占比等
- 交大各学科的传播媒介分布（仅交大做此统计）

**人机分工：**
- AI：统计分析已爬取内容的呈现方式分布
- 人：确认统计口径和结论解读

---

### 阶段C：企业轨

#### C1. 企业视觉案例采集

**产出：**
- 确定对标企业清单：追觅、小米、苹果、Tesla、Boston Dynamics、SpaceX等
- 采集企业视觉素材：官网产品页、发布会物料、技术博客配图、社交媒体传播图
- 入库 Sci-Viz Case Hub（标注 `source_type: enterprise`）

**人机分工：**
- 人：确定企业清单和采集优先级
- AI：编写企业来源爬虫适配器
- AI：自动采集+AI分类入库
- 人：复核企业案例质量

#### C2. 企业视觉商业化路径分析

**产出：**
- 每个企业的视觉表达策略梳理
- 企业如何将技术/研究转译为视觉故事的分析
- 学术机构与企业视觉表达的差异总结

**人机分工：**
- AI：辅助整理策略特征、视觉风格统计
- 人：做审美和策略判断

---

### 阶段D：整合分析（A、B、C完成后进行）

#### D1. 三栏/四栏对比分析

按交大学科维度，做四栏对比：

| 模块 | 内容 | 数据来源 |
|---|---|---|
| 交大现状 | 交大某学科的呈现方式、比例、问题 | 阶段B |
| 国内研究 | 同学科国内其他高校/机构标杆 | 学术轨 |
| 国际研究 | 同学科国际顶刊/机构标杆 | 学术轨 |
| 头部企业 | 该领域相关企业的视觉商业化 | 阶段C |

在 Sci-Viz Case Hub 中实现四栏对比视图（系统功能）。

#### D2. 各类型子分类交叉分析

按三轴做交叉分析：
- 功能×技术：不同功能用途偏好什么技术手段？
- 功能×媒介：不同功能用途通过什么媒介传播？
- 技术×媒介：不同技术手段产出什么媒介形式？

#### D3. 三轴交叉图谱

所有案例按三轴标注后，生成交叉图谱/散点图，可视化呈现各学科、各来源在三轴空间中的位置。

---

### 阶段E：系统内总结

在 Sci-Viz Case Hub 中增加总结/洞察模块，不需要单独导出报告：

- 四栏对比视图
- 学科→顶刊映射表
- 各类型子分类分布
- 三轴交叉图谱
- 交大呈现方式比例概览
- 企业视觉商业化路径概览

## 四、与现有阶段0-6的关系

现有 `视觉调研流程_人机分工.md` 中的阶段0-6（编码本→筛选→编码→分析→谱系→拆解→报告）属于**学术轨**，继续推进不变。

新增工作融入方式：

| 新增内容 | 融入位置 |
|---|---|
| 三轴分类体系 | 加入阶段0编码本开发 |
| 子分类字典 | 加入阶段0编码本开发 |
| 数据库字段变更 | 编码本定稿后即可实施 |
| 交大深研 | 新增并行轨道B |
| 企业轨 | 新增并行轨道C |
| 四栏对比 | 阶段3分析时整合 |
| 系统内总结 | 替代原阶段6报告的部分功能 |
