# 科研影像自动收集信息进度汇报

汇报日期：2026 年 5 月 31 日  
项目方向：科研影像前期调研与工作室建设  
当前阶段：自动采集与案例库地基已搭建，素材库已形成初步规模，下一步进入视觉编码、人工复核与策略分析阶段

---

## 一、总体进展概述

根据项目要求，本次调研不只是“找图片”，而是要为后续科研影像咨询工作室建立一套可复用的视觉策略知识体系，包括国际基准分析、风格图谱、渠道研究、竞品分析、案例库与交付模板。

目前已经完成了自动化收集系统的主体搭建，并初步形成了“来源池 + 自动采集 + 图片入库 + AI/规则分类 + 人工复核 + 导出”的工作流。系统名称为 **Sci-Viz Case Hub**，当前已具备科研视觉案例的集中管理、来源追踪、分类统计、去重、缩略图生成、复核和导出能力。

截至目前，数据库中已有 **2837 条科研视觉案例记录**，另有本地期刊封面素材库 **1830 张原始封面图**，覆盖 Nature 及多个 Nature 子刊、NASA、MIT、Harvard、Berkeley Lab、Max Planck，以及中科院、清华、北大、复旦、上海交通大学、中国科学技术大学等来源。整体上，自动化采集系统已从概念验证进入可持续扩展阶段。

## 二、目前已建设的资料库

### 1. 结构化案例数据库

当前核心数据库为 `sci-viz-case-hub/server/prisma/dev.db`，采用 SQLite + Prisma 管理。数据库已包含 4 类核心表：

| 数据表 | 用途 |
|---|---|
| `VisualCase` | 科研视觉案例主表，保存图片、来源、上下文、分类、AI 摘要、复核状态、评分等信息 |
| `CrawlSource` | 来源池，管理可采集网站、来源类型、采集状态、分层优先级 |
| `CrawlJob` | 采集任务记录，保存发现链接数、已采集数量、新增案例数、失败原因等 |
| `CollectionKpi` | 分类配额表，用于判断不同媒介、学科、风格、内容类型的覆盖情况 |

目前 `VisualCase` 已入库 **2837 条**，状态均已标记为 `approved`，说明当前素材已经完成了系统层面的入库整理，后续重点是进一步人工筛选高价值案例、补充分析笔记和方法论归纳。

### 2. 本地图片素材库

目前本地已经形成两类素材库：

| 素材库 | 当前规模 | 说明 |
|---|---:|---|
| `journal_covers/` | 1830 张原始期刊封面图 | 覆盖 Nature 及多个 Nature 子刊，另有缩略图，总文件量约 3660 张 |
| `nature_covers/` | 17 张 Nature 2026 年近期封面 | 作为单独脚本早期采集成果 |
| `server/uploads/originals/` | 994 个系统入库原图文件 | 后端自动下载和保存的原始图片 |
| `server/uploads/thumbnails/` | 544 个系统生成缩略图 | 用于前端快速浏览 |

期刊封面库目前是最成规模的一类材料，适合用于分析顶刊封面图像的媒介类型、色彩策略、科学叙事方式和学科差异。

## 四、自动收集工作流进展

目前已经形成以下自动化流程：

```text
来源池管理
  → 列表页链接发现
  → 文章页/详情页抓取
  → 图片候选提取
  → 图片过滤与评分
  → 图片下载与本地保存
  → 缩略图生成
  → 精确去重
  → OCR / 视觉 AI 分析
  → 结构化入库
  → 人工复核
  → CSV / Markdown 导出
```

### 1. 来源池管理

系统已实现来源池页面和后端接口，可以按来源类型管理不同网站。每个来源包含名称、URL、来源类别、视觉价值说明、采集策略、采集状态和优先级。

已支持的来源状态包括：

| 状态 | 含义 |
|---|---|
| `active_static` | 静态 HTML 可采集 |
| `easy_static` | 已验证、适合小批量自动采集 |
| `active_pw` | 需要 Playwright 浏览器渲染采集 |
| `needs_adapter_tuning` | 可访问，但需要专门适配器调优 |
| `blocked_cloudflare` | 被 Cloudflare 或强反爬拦截 |
| `manual` | 暂时采用人工或插件采集 |

### 2. 链接发现

已实现从列表页自动发现文章链接的功能，核心逻辑包括：

- 识别 `/news/`、`/article/`、`/story/`、日期路径等文章 URL 特征；
- 根据链接文本长度、位置、语义进行评分；
- 排除导航、页脚、登录、分享、PDF、邮件链接等低价值链接；
- 支持分页遍历；
- 支持去重和按分数排序。

这一步解决了过去需要人工逐个复制文章 URL 的问题。

### 3. 图片提取与过滤

已实现文章页图片候选提取，包括：

- 普通 `img`；
- `srcset` 中高分辨率图片；
- 懒加载 `data-src`；
- `figure` / `figcaption` 上下文；
- 附近段落文本；
- `og:image`、`twitter:image` 等页面主图。

同时已实现低价值图片过滤：

- 排除 logo、icon、avatar、sprite、广告图、像素图；
- 排除过小图片；
- 排除 nav/header/footer/form/comment/cookie 等区域中的图片；
- 对科研关键词、caption、上下文、图片尺寸和来源可信度进行候选评分。

### 4. 下载、去重与保存

后端已实现图片安全下载与处理：

- 只允许公网 HTTP/HTTPS URL；
- 拦截私网、localhost、DNS 指向私网等风险地址；
- 限制响应体大小；
- 校验 content-type 和真实图片格式；
- 保存原图并生成缩略图；
- 使用 SHA-256 做精确去重；
- 使用感知哈希作为未来相似图人工检查的辅助字段。

当前自动去重策略是保守的：只有完全相同的图片才自动跳过，相似图不自动删除，避免误删科研图像中有价值的不同裁切、不同版本或同类实验图。

### 5. AI / OCR 分析

系统已预留并实现 OCR 与视觉 AI 分析接口：

- OCR 服务可通过环境变量配置；
- 视觉分析服务基于 OpenRouter / Qwen VL 方案；
- 自动输出媒介类型、学科、视觉风格、构图、色调、用途、摘要、可借鉴点、风险提示和置信度；
- 当 API 未配置时，系统可返回 mock 结果，保证流程不中断。

目前数据库中已经形成了第一版自动分类结果，但后续仍需要进行人工抽样校验，尤其是“风格”“学科”“内容类型”等标签需要进一步标准化。

## 八、当前不足与风险

### 1. 数据质量仍需人工复核

当前系统已经可以自动采集和自动分类，但科研影像调研最终需要形成“可引用、可解释、可借鉴”的高质量案例。自动分类只能作为初筛，后续需要人工复核：

- 删除低价值图；
- 修正学科和风格标签；
- 标注可借鉴点；
- 评 4-5 分高价值样本；
- 为典型案例补充创作背景和传播效果。

### 2. 样本结构有偏

当前样本中顶刊封面占比较高，优势是便于分析顶刊审美和科学叙事；不足是实验室真实场景、科研人员肖像、团队合影、设备空间、国内服务市场案例仍偏少。下一阶段需要有意识补齐“工作室未来服务场景”相关样本。

### 3. 部分重要来源受反爬限制

Stanford、Science、部分图库和动态网页存在 Cloudflare、JS 动态加载、网络访问或 API 问题。当前策略是不硬破解，而是将来源标记为 blocked/manual/needs_adapter_tuning，后续用 API、Playwright 或浏览器插件补采。

### 4. 文献与市场调研尚未系统化

目前自动收集重点放在视觉案例和技术系统上，项目要求中的文献研究、国内服务市场、价格/商业模式、工作室交付模板还需要单独推进。

## 十、给老师的简短汇报口径

目前我们已经完成了科研影像自动收集系统的第一阶段建设。系统已经不是简单文件夹式收图，而是建立了包含来源池、自动采集、图片去重、AI/规则分类、人工复核和导出的结构化案例库。

截至目前，案例数据库中已有 2837 条科研视觉案例，另有 1800 多张 Nature 系列期刊封面原始图。来源覆盖 Nature 及多个子刊、NASA、MIT、Harvard、Berkeley Lab、Max Planck，以及中科院、清华、北大、复旦、上海交大、中科大等国内外科研机构和高校。系统已验证 MIT、Harvard、Berkeley Lab、清华、北大、中科院等来源可以自动化采集；Stanford、Science、CERN 等来源已识别出技术限制，后续将通过专门适配器或人工插件方式补充。

对照原项目要求，目前“素材采集、来源池建设、案例库地基、期刊封面样本、国内外高校科研新闻样本”已经完成第一阶段；“视觉编码本、人工复核、风格图谱、典型案例拆解、文献研究、国内市场调研、最终策略报告”是下一阶段重点。整体进度已经从工具搭建进入到知识整理和策略分析阶段。
