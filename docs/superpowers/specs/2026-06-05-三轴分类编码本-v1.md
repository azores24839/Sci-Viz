# 三轴分类编码本 v1

> 版本：v1.0 | 日期：2026-06-05 | 状态：待人工复核
> 本文档定义科研影像"功能—媒介—技术"三维分类体系，用于标注、统计和诊断。

---

## 一、总览：三轴是什么？

每一条科研影像都可以从三个独立的角度来理解：

| 轴 | 核心问题 | 类别 |
|----|---------|------|
| **功能** | 这张影像是用来干什么的？ | 记录、解释、数据、展示、传播、交互 |
| **媒介** | 这张影像以什么形式存在？ | 静图、动图、视频、图组、交互、实体 |
| **技术** | 这张影像是怎么生产出来的？ | 拍摄、成像、绘设、数据、渲染、生成 |

三个轴相互独立。不能写成"记录30%＋视频40%＋渲染30%"——因为它们不是同一层级的东西。正确做法是分别给出三套比例。

## 二、轴一：功能维度（这张影像是用来干什么的？）

### 2.1 类别定义

| 代码 | 类别 | 定义 | 典型例子 |
|------|------|------|---------|
| REC | 记录 | 保存真实对象、事件、人物或过程的影像 | 实验现场照片、设备外观、人物合影、实验室环境 |
| EXP | 解释 | 说明复杂机制、结构、流程和原理的影像 | 工作原理图、系统结构图、实验流程图、原理示意图 |
| DAT | 数据 | 呈现研究结果、模型和变化趋势的影像 | 性能对比图、统计图表、模型输出、趋势曲线 |
| SHO | 展示 | 呈现成果、产品、空间或项目形象的影像 | 产品照片、成果外观、空间展示、项目主视觉 |
| OUT | 传播 | 面向公众、品牌和科普传播的影像 | 科普图、新闻宣传、封面、招生海报 |
| INT | 交互 | 支持用户操作、浏览、筛选和决策的影像 | 数据仪表盘、交互地图、可旋转3D模型 |

### 2.2 每个类别的详细判断规则

#### REC 记录
- **核心判断**：影像是否在"如实保存"某个真实存在的东西？
- **属于记录**：设备外观照、实验过程抓拍、实验室环境、人物肖像、会议现场
- **不属于记录**：经过设计的信息图（→解释）、有明确宣传意图的封面图（→传播）
- **容易混淆的情况**：
  - 一张科研人员的照片：如果是为了展示团队形象（→传播），如果只是存档（→记录）
  - 实验过程的视频截图：如果是记录过程本身（→记录），如果是解释实验步骤（→解释）

#### EXP 解释
- **核心判断**：影像是否在帮你理解"为什么""怎么来的""结构是什么"？
- **属于解释**：机制原理图、系统架构图、流程步骤图、因果关系图、结构剖析图
- **不属于解释**：单纯的数据曲线（→数据）、产品外观展示（→展示）
- **容易混淆的情况**：
  - 带有标注的产品渲染图：如果标注是为了说明结构（→解释），如果只是为了好看（→展示）
  - 流程图：都是解释类，不是数据类——即使流程里有数据节点

#### DAT 数据
- **核心判断**：影像的核心信息是否是数字、趋势、比较结果？
- **属于数据**：实验结果图、统计柱状图、折线趋势、热力图、散点图、模型模拟输出
- **不属于数据**：说明数据收集方法的流程图（→解释）
- **容易混淆的情况**：
  - 信息图里的数据组：如果重点是数字关系（→数据），如果重点是过程或故事（→解释或传播）
  - 带数据的地图：如果是用数据着色展示空间分布（→数据），如果只是标注位置（→记录）

#### SHO 展示
- **核心判断**：影像的核心目的是不是"看，我们做出了这个东西"？
- **属于展示**：机器人运行截图、新材料外观、建筑效果图、产品渲染图
- **不属于展示**：同一产品但标注了结构说明的（→解释）
- **容易混淆的情况**：
  - 渲染图：纯粹的静物渲染（→展示），标注了内部结构的剖视图（→解释）
  - 实验设备照片：只是展示设备长什么样（→记录也可能是展示），如果强调"这是我们做出的成果"（→展示）

#### OUT 传播
- **核心判断**：影像是否主要面向非专业受众，目的是让更多人知道或理解？
- **属于传播**：Nature封面图、科普信息图、项目宣传海报、新闻配图、招生图片
- **不属于传播**：给同行看的论文图表（→数据）、只给自己团队存档的（→记录）
- **容易混淆的情况**：
  - Nature封面图：主功能是传播（→传播），即使图片本身可能是渲染或显微
  - 科研成果的新闻稿配图：传播（→传播），不是展示
  - 一个项目主页的主视觉图：既有展示功能也有传播功能——取主要意图

#### INT 交互
- **核心判断**：用户能否主动操作来获取不同信息？
- **属于交互**：数据仪表盘、可缩放地图、可旋转3D模型、筛选器面板
- **不属于交互**：交互界面的截图（→记录或展示）
- **容易混淆的情况**：
  - 视频中的交互式演示录像：录像本身是视频（→视频/记录），交互内容是交互（→交互）
  - 可点击的图组：如果只能前后翻页（→图组），如果能根据筛选动态变化（→交互）

### 2.3 正例与反例

| 影像描述 | 正确归类 | 常见误判 |
|---------|---------|---------|
| Nature封面图 | 传播 | 误判为展示 |
| 机器人运行视频 | 展示（核心是"看成果"） | 误判为记录 |
| 设备外观照片 | 记录 | 可能同时是展示——取主要意图 |
| 性能对比柱状图 | 数据 | 误判为解释 |
| 工作原理动画 | 解释 | 误判为展示 |
| 可交互的数据地图 | 交互 | 无——边界较清楚 |
| 团队合影 | 记录 | 误判为传播 |
| 科普信息图 | 传播 | 无——边界较清楚 |

## 三、轴二：媒介维度（这张影像以什么形式存在？）

### 3.1 类别定义

| 代码 | 类别 | 定义 | 典型例子 |
|------|------|------|---------|
| STA | 静图 | 单张静态图像 | 摄影照片、插图、图表、渲染图 |
| ANI | 动图 | 带有简单运动或循环效果的图像 | GIF演示、循环动效、局部动画 |
| VID | 视频 | 连续影像内容 | 实验视频、讲解视频、动画视频 |
| SET | 图组 | 多张图像有逻辑地组合在一起 | 步骤图、对比图组、信息图长图 |
| INT | 交互 | 可被用户操作和探索的视觉界面 | 仪表盘、交互地图、3D查看器 |
| PHY | 实体 | 转化到现实空间中的视觉媒介 | 海报、展板、印刷刊物、展览 |

### 3.2 每个类别的详细判断规则

#### STA 静图
- **核心判断**：是一张固定不变的图片吗？
- **属于静图**：任何单帧图像（JPG、PNG等）
- **不属于静图**：GIF动图（→动图）、网页仪表盘截图（截图本身是静图，但被截图的内容是交互）

#### ANI 动图
- **核心判断**：图片自己能动，但不是完整视频？
- **属于动图**：GIF、循环Web动效、CSS动画截图（如果动效是核心内容）
- **不属于动图**：短视频（→视频）、静图加后期动效（如果原内容是静图→静图）

#### VID 视频
- **核心判断**：是有时间轴的连续影像，时长通常>几秒？
- **属于视频**：实验录像、动画短片、采访视频、演示视频
- **容易混淆**：短视频（10秒以内）可能是动图也可能是视频，看是否有完整叙事结构

#### SET 图组
- **核心判断**：多张图按逻辑编排在一起，各有分工？
- **属于图组**：Nature的multi-panel figure、步骤对比图、论文中的Figure 1(a)(b)(c)
- **不属于图组**：多张图只是排列在一起没有逻辑关系（→多张静图）

#### INT 交互
- **核心判断**：用户能否操作改变显示内容？
- **属于交互**：Tableau仪表盘、可缩放地图、可旋转3D模型
- **容易混淆**：交互内容的截图（→静图）

#### PHY 实体
- **核心判断**：是否以物理形态展示？
- **属于实体**：学术海报、展板、印刷宣传册
- **当前数据库暂无此类型案例**

### 3.3 当前数据库字段映射

| 数据库字段 | 对应关系 |
|-----------|---------|
| `distributionMedium` | **直接对应**此轴，值域为：静图/动图/视频/图组/交互/实体 |
| `captureType`="video" | 映射到 视频 |
| `videoUrl`非空 | 强指示为 视频 |
| 默认值 | 目前98%以上被回填为 静图 |

**注意**：当前数据中 `distributionMedium` 极度偏斜——94.7%为"静图"，只有26条非静图。这一偏斜需要通过采集更多视频、动图、交互内容来纠正，而非把它当作"静图为主"的结论。

## 四、轴三：技术维度（这张影像是怎么生产出来的？）

### 4.1 类别定义

| 代码 | 类别 | 定义 | 典型例子 |
|------|------|------|---------|
| PHO | 拍摄 | 通过相机、摄像机等获取现实影像 | 实验现场摄影、设备照片、航拍 |
| IMG | 成像 | 通过专业设备获取不可见或微观影像 | 光学显微、电镜、CT/MRI、遥感、热成像 |
| DES | 绘设 | 通过人工绘制和视觉设计方法生成 | 科学插画、图标设计、信息图排版、流程图手绘 |
| DAT | 数据 | 通过数据处理与可视化生成 | 统计图表、数据地图、网络图、科学数据可视化 |
| REN | 渲染 | 通过三维建模、仿真和工程软件生成 | 3D产品渲染、3D机制图、3D场景、仿真动画 |
| GEN | 生成 | 通过AI或算法增强、转化 | AI生成图像、算法增强、风格迁移 |

### 4.2 每个类别的详细判断规则

#### PHO 拍摄
- **核心判断**：是"镜头对准真实物体"拍出来的吗？
- **属于拍摄**：实验现场照、设备外观照、人物肖像、航拍
- **不属于拍摄**：显微镜照片（→成像）、3D渲染输出（→渲染）
- **容易混淆**：
  - 航拍照片：属于拍摄（→拍摄），不是成像
  - 经过后期的照片：后期只是修饰，核心仍是拍摄（→拍摄）

#### IMG 成像
- **核心判断**：是否使用了"超越肉眼"的专业成像设备？
- **属于成像**：光学显微、电子显微（SEM/TEM）、医学影像（CT/MRI/X光/超声）、遥感成像、热成像
- **不属于成像**：普通近拍/微距（→拍摄）、3D模拟的内部结构（→渲染）
- **容易混淆**：
  - NASA天文照片：大部分经过后期处理的属于成像（→成像），但如果是概念渲染（→渲染）
  - 超声视频：既是成像又是视频——技术维度判为成像，媒介维度判为视频

#### DES 绘设
- **核心判断**：是否主要由人工绘制、设计、排版而成？
- **属于绘设**：科学插画、图标、流程图、信息图、版面设计
- **不属于绘设**：3D渲染图（→渲染）、数据图表（→数据）
- **容易混淆**：
  - 信息图：如果包含大量数据图表为核心（→数据），如果以插画和排版为核心（→绘设）
  - 手绘风格的渲染图：核心生产技术是3D渲染（→渲染），手绘只是风格

#### DAT 数据
- **核心判断**：核心信息是否通过数据处理和可视化算法呈现？
- **属于数据**：柱状图、折线图、热力图、数据地图、网络关系图
- **不属于数据**：数据收集过程的照片（→记录/拍摄）
- **容易混淆**：
  - 带数据的信息图：数据是核心（→数据），图解是辅助（→绘设）——取主要技术手段
  - 简单的示意图：如果是手绘/设计（→绘设），如果是数据驱动（→数据）

#### REN 渲染
- **核心判断**：是否通过三维建模/仿真/工程软件生成？
- **属于渲染**：3D产品渲染、3D机制图、仿真场景、CAD输出、工程动画帧
- **不属于渲染**：2D手绘的结构图（→绘设）、真实照片（→拍摄）
- **容易混淆**：
  - 2D风格的3D渲染：即使看起来扁平，只要核心是建模渲染的（→渲染）
  - 渲染图+后期设计：核心生产技术是渲染（→渲染），后期处理不算改变类别

#### GEN 生成
- **核心判断**：是否主要依靠AI或算法生成/增强？
- **属于生成**：Diffusion模型输出、GAN生成图、算法风格迁移、AI增强超分
- **不属于生成**：使用AI辅助修图的照片（→拍摄）、使用AI辅助标注的显微图（→成像）
- **容易混淆**：
  - AI辅助3D建模：核心仍是建模渲染（→渲染），AI是辅助不算改变类别
  - 生成式数据可视化：核心是数据处理（→数据），算法只是手段
  - 当前数据库只有4条"生成"——这并不意味着学术界不用AI，而是目前采集来源中AI生成的影像极少

## 五、数据库字段映射表

### 5.1 现有字段 → 三轴映射

| 数据库字段 | 对应哪一轴 | 当前值域 | 映射说明 |
|-----------|-----------|---------|---------|
| `functionalPurpose` | **功能轴** | 记录/解释/数据/展示/传播/交互 | 直接对应，无需转换 |
| `distributionMedium` | **媒介轴** | 静图/动图/视频/图组/交互/实体 | 直接对应，无需转换 |
| `technicalMethod` | **技术轴** | 拍摄/成像/绘设/数据/渲染/生成 | 直接对应，无需转换 |
| `mediaType` | 技术+媒介混合 | 摄影/手绘图/3D渲染/信息图/显微图/数据可视化/混合媒介/不确定 | **混淆了技术和媒介**。需要拆分：摄影→(技术=拍摄, 媒介≈静图)，3D渲染→(技术=渲染, 媒介≈静图)，信息图→(技术=绘设, 媒介≈静图或图组) |
| `contentType` | 功能+内容混合 | 单人肖像/群体肖像/…/不确定 | **混淆了功能和内容对象**。如"科普传播"是功能（→传播），"实验设备"是内容对象（→记录） |
| `discipline` | 独立维度 | 10个学科类别 | 不属于三轴，是分析维度 |
| `mediaSubType` | 技术子分类 | 如"纪实摄影""3D机制图""科学插画"等 | 对应技术轴子分类 |
| `contentSubType` | 功能子分类 | 如"科普传播""机制图解""群体团队"等 | 对应功能轴子分类 |

### 5.2 需要注意的混淆点

| 字段 | 混淆原因 | 建议 |
|------|---------|------|
| `mediaType` | 把技术（渲染）和视觉风格（手绘图）混在一起，且不区分媒介形式 | 已拆分为三轴，`mediaType`仅作参考 |
| `contentType` | 混合了功能（科普传播→传播）和内容对象（实验设备→记录） | 已用`functionalPurpose`替代功能部分 |
| `rating` | 当前几乎无效（1847条=5星，2043条未评），不能用于区分"高质量" | 需重建评价机制，当前不可用于对标分析 |
| `confidence` | 呈两极分布（49%<0.5, 51%>0.8），中间值极少 | 可用于AI标注置信度，不能替代人工质量判断 |

### 5.3 回填脚本的局限

目前三轴数据主要通过规则回填（`backfillThreeAxis.ts`）获得，而非AI重新分析。回填规则较粗：

| 规则 | 问题 |
|------|------|
| 功能轴：technicalMethod=绘设 → 功能=传播 | 不准确——绘设可以是解释图（→解释），不一定都是传播 |
| 功能轴：technicalMethod=拍摄 → 功能=记录 | 不准确——产品拍摄可以是展示（→展示） |
| 功能轴：technicalMethod=渲染 → 功能=展示 | 部分准确，但渲染也可以是解释（→解释） |
| 媒介轴：默认全部静图 | **严重偏斜**——94.7%为静图，实际应包含更多视频/图组/交互 |
| 媒介轴：captureType=video → 视频 | 合理，但只识别了23条 |

**结论**：回填数据只能用于初步统计方向判断，不能当作精确标注。需要AI重新分析或人工标注来校正。

## 七、后续人工标注注意事项

1. **先定功能，再看媒介和技术**：功能维度最需要人工判断，因为"这张图是用来干什么的"最容易产生歧义。

2. **一个影像只有一个主功能**：如果一张图同时有记录和展示功能，选择你判断的"主要意图"。但可以在子分类中备注次要功能。

3. **媒介判断先看形式，不看内容**：一张照片内容再丰富，媒介维度就是"静图"。一段视频内容再无聊，媒介维度也是"视频"。

4. **技术判断看"核心生产方式"**：如果一张图经过多种技术手段处理，选择"决定最终视觉效果的核心技术"。比如：3D渲染后用PS排版，核心技术是渲染（→渲染）。

5. **现有回填数据的490条空功能值和180条空媒介值需要优先补全**。

6. **"传播"被严重高估（1896条，48.5%），"展示"和"交互"被严重低估（35条和0条）**——这是回填规则的系统性偏差，需要人工校正。

7. **不要用rating字段来判断质量**——当前rating几乎无效（只有5星和0星两个值，中间值极少），需要重建评价机制。

8. **交互类型目前案例数为0**——这是采集来源的偏差（学术网站交互内容少），不意味着交互不重要。需要补充交互式数据可视化来源。
