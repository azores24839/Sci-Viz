# Sci AI Studio 资料上传与解析设计规格

> 日期：2026-07-02  
> 状态：已完成产品讨论，待实施  
> 适用范围：`sci-viz-studio/`

## 1. 目标

为 Sci AI Studio 建立统一、可持久化、可追溯的资料池。用户可以主动提供 PDF、Word、PNG/JPG、粘贴文字和公开网页链接；系统完成保存、解析、OCR、AI 摘要和选择管理，再把选中的真实资料交给“资料分析师”及后续工作流使用。

本阶段只实现用户主动提供资料，不实现 Fast Research、Deep Research、自动全网搜索或递归爬取。

## 2. 产品原则

1. 原始资料、提取文字和 AI 摘要分层保存，AI 结果不覆盖原文。
2. 文档文字层和网页正文解析不依赖 AI；图片与扫描页 OCR 使用 Qwen。AI 不可用时，原文件与已经完成的基础解析仍被保留并可重试。
3. 用户决定哪些资料进入分析。上传成功不等于自动参与后续 Agent 工作。
4. 每条结论保留来源标识；网页保留原网址和抓取时间，文档保留页码或结构位置。
5. 资料失败时给出可理解原因和重试入口，不静默丢失内容。
6. Studio 独立实现资料服务，不在运行时依赖 Case Hub；只复用其 Qwen/DashScope 配置思路和同一份用户密钥。
7. 生产环境以阿里云 ECS + 私有 OSS + Linux Docker 为基线，不依赖 macOS、Swift 或开发者电脑常驻。

## 3. 支持范围

### 3.1 文件与输入类型

| 类型 | 首版支持 | 处理方式 |
| --- | --- | --- |
| PDF | 是 | 提取文字层；文字不足时逐页 OCR；保留页码 |
| DOCX | 是 | 提取标题、段落和表格文字 |
| DOC | 否 | 提示用户转换为 DOCX 或 PDF |
| PNG/JPG/JPEG | 是 | 保存原图、生成缩略图、Qwen OCR、图片描述与摘要 |
| 粘贴文字 | 是 | 保存原文，自动生成标题和摘要 |
| 公开网页 URL | 是 | 抓取单页正文、标题、站点和时间，再生成摘要 |
| PDF URL | 是 | 下载后按 PDF 处理 |
| 登录网页、付费墙、递归网页 | 否 | 给出明确的不可解析提示 |

### 3.2 默认限制

- 单文件最大 50 MB。
- 单项目最多 50 份资料。
- 网页下载体积最大 20 MB。
- 网页与粘贴文字最多保留 50,000 个中文字符或等量文本；超出部分截断并在界面提示。
- 网页最多跟随 5 次重定向，每次跳转都重新执行安全检查。
- 文档与 OCR 处理设置明确超时；超时后进入可重试失败状态。

这些是 MVP 保护性限制，不作为长期商业额度承诺。

## 4. 用户流程

```text
添加资料
  → 服务端保存原文件 / 原文 / 原始网址
  → 类型识别与安全校验
  → Worker 执行跨平台解析或 Qwen OCR
  → Qwen 生成摘要与图片描述
  → 资料进入“已就绪”或可重试状态
  → 用户查看、勾选、取消勾选、重试或删除
  → 至少一份已就绪且已勾选的资料后，才可进入视觉现状诊断
```

资料状态统一为：

- `UPLOADING`：上传中。
- `QUEUED`：已保存，等待解析。
- `PARSING`：正在提取文字或 OCR。
- `SUMMARIZING`：基础解析完成，正在生成 AI 摘要。
- `READY`：基础解析和摘要均完成。
- `READY_WITHOUT_SUMMARY`：基础解析完成，AI 摘要失败，可继续使用或单独重试摘要。
- `FAILED`：基础解析失败，保留原始资料和错误原因，可重试。

## 5. 界面设计

### 5.1 流程节点

“资料输入”节点只显示摘要信息：资料总数、已选数量、已完成数量、失败数量和整体状态。节点卡不承载完整上传表单，避免画布拥挤。

### 5.2 右侧资料面板

右侧面板提供四个入口：

1. 上传文档：PDF、DOCX。
2. 上传图片：PNG、JPG、JPEG。
3. 粘贴文字：标题可选、正文必填。
4. 添加链接：单个公开网页或 PDF 链接。

每条资料显示：

- 类型、名称、大小或网址。
- 当前解析状态和进度说明。
- 是否选入本轮分析。
- AI 摘要，并明确标记“AI 生成”。
- 原文/提取文字预览；图片额外显示缩略图、OCR 文字和 AI 图片描述。
- 查看、勾选、取消勾选、重试、重新生成摘要和删除操作。
- 失败原因、截断提示、抓取时间和来源地址。

进入下一步的门槛：至少一份资料状态为 `READY` 或 `READY_WITHOUT_SUMMARY`，且被用户勾选。

## 6. 数据模型

新增统一 `SourceDocument` 契约，核心字段包括：

- `id`、`projectId`。
- `kind`：`PDF | DOCX | IMAGE | TEXT | WEB`。
- `status`、`selected`。
- `title`、`originalName`、`mimeType`、`sizeBytes`。
- `sourceUrl`、`canonicalUrl`、`fetchedAt`。
- `storagePath`、`thumbnailPath`。
- `contentHash`。
- `rawText`、`extractedText`、`ocrText`。
- `aiSummary`、`imageDescription`、`summaryProvider`、`summaryModel`。
- `pageCount`、`truncated`。
- `errorCode`、`errorMessage`。
- `createdAt`、`updatedAt`。

解析结果按页或结构块保留来源定位，以便后续 Agent 输出引用到 PDF 页码、DOCX 段落或网页正文。

生产环境使用持久数据库保存元数据与任务状态，原文件和缩略图存入私有 OSS；本地开发使用同一接口的本地存储适配器。页面刷新、服务重启和 Worker 重启后，资料列表、选择状态和解析结果仍存在。

## 7. 服务边界与数据流

### 7.1 资料服务

服务端新增独立资料模块，职责拆分为：

- `sourceRepository`：元数据与状态持久化。
- `sourceStorage`：统一存储接口；提供本地开发适配器和阿里云 OSS 生产适配器，负责安全对象键、原文件、缩略图、签名预览和删除。
- `sourceParser`：按类型路由到 PDF、DOCX、图片、文字或网页解析器。
- `ocrService`：通过 Qwen OCR 处理图片和扫描页；失败时返回可诊断错误。
- `webSourceFetcher`：URL 校验、下载、正文提取和规范化。
- `sourceSummaryService`：通过 Qwen 生成摘要或图片描述。
- `sourceJobRunner`：持久任务队列和独立 Worker；上传请求只创建任务，不在 HTTP 请求内完成重解析。

模块通过明确 DTO 通信，不把 HTTP、文件系统、解析库和 AI 调用混进同一个文件。

### 7.2 API

首版提供以下能力：

- 上传一个或多个文件。
- 创建粘贴文字资料。
- 创建网页链接资料。
- 获取项目资料列表与单条详情。
- 更新是否选中。
- 重试基础解析。
- 重新生成 AI 摘要。
- 删除资料及其本地文件。
- 安全读取缩略图或原文件预览。

接口使用共享 Zod 契约校验输入和输出。前端只显示服务端返回的安全文件标识，不接触真实磁盘路径。

### 7.3 Agent 输入

资料分析师请求不再只使用 Mock 资料，而是读取项目中已勾选且基础解析成功的资料：

- 文档、网页和粘贴文字提供提取正文及来源定位。
- 图片提供 OCR、AI 图片描述和安全的图片引用。
- 长文本按来源切分并限制总上下文；任何截断必须进入 Agent 输入说明。
- Mock 资料仍可作为明确标记的演示回退，但不能与真实资料混淆。

## 8. 解析策略

### 8.1 PDF

先读取文字层并记录页码。如果整份文档提取文字过少或多数页面为空，则由 Linux/Docker 中的 Poppler 将相应页面转为图片，再调用 Qwen OCR。单页失败不阻塞其他页面，最终结果标出失败页。

### 8.2 DOCX

提取标题、段落、列表和表格文字，保持原始顺序。图片嵌入内容首版不单独执行 OCR。遇到损坏文档时保留文件并返回可理解错误。

### 8.3 图片

验证真实 MIME 和像素尺寸，生成缩略图；Qwen OCR 提取可见文字，并结合用户标题生成客观图片描述与资料摘要。不确定内容必须标记“待确认”，不得补写科研事实。

### 8.4 粘贴文字

原样保存用户输入，标准化换行用于解析。标题为空时生成短标题；AI 摘要失败不影响原文使用。

### 8.5 网页

仅允许 `http` 和 `https`。抓取前解析 DNS，拒绝 localhost、回环地址、私网地址、链路本地地址、保留地址和云元数据地址；每次重定向重复校验。只提取当前页面正文，不导入嵌入视频和下级页面。若响应是 PDF，则转入 PDF 流程。

## 9. AI 与密钥

- Qwen/DashScope 仅在 Studio 服务端调用。
- 使用 `DASHSCOPE_API_KEY`、明确的兼容 API 地址和模型配置；不得把 DashScope key 发送到 OpenRouter 地址。
- 可以使用与 Case Hub 相同的密钥值，但 Studio 的运行不读取 Case Hub 代码或前端配置。
- 密钥只存在于未跟踪的 `.env` 或进程环境中，日志不得打印完整密钥、Authorization 头或原始敏感资料。
- AI 摘要包含来源类型和有限正文；图片理解包含图片本身和 OCR 结果。
- AI 请求失败、限流或额度耗尽时，资料进入 `READY_WITHOUT_SUMMARY`。
- OCR 与摘要任务通过 Worker 执行，使用有限并发、指数退避和最大重试次数，避免 20 名测试用户同时上传时击穿模型配额。

## 9.1 生产部署基线

- Web 构建产物由 OSS 静态托管或 ECS 上的 Nginx 提供。
- 浏览器通过服务端签发的短期凭证或签名请求直传私有 OSS，50 MB 文件不经过应用服务器内存。
- API 与 Worker 以两个 Docker 进程运行在 ECS；Worker 可单独扩容。
- 元数据和任务状态保存在 PostgreSQL；早期单机部署可将 PostgreSQL 放在同一 ECS 并每日备份，稳定后迁移 RDS。
- OSS、ECS、PostgreSQL 与 DashScope 优先使用同一地域和内网访问路径。
- 本地开发允许使用本地文件存储与开发数据库，但生产启动时若仍为本地适配器必须输出明确警告。

## 10. 去重与删除

- 本地文件使用内容哈希去重；重复时提示用户复用已有资料，不重复存储。
- 网页使用规范化 URL 检查重复；去除常见跟踪参数，但保留可能影响正文的业务参数。
- 删除先移除资料引用，再删除不再被任何资料引用的本地文件。
- 删除操作需要用户确认；失败时不得让数据库状态与文件状态悄悄分裂。

## 11. 错误处理与隐私

用户可见错误使用可操作文案，例如“文件超过 50 MB”“旧版 DOC 暂不支持”“网页需要登录”“网页地址指向内网”“扫描 PDF 第 4 页 OCR 失败”。内部错误码保留用于测试和日志。

上传资料可能包含未公开科研信息。生产文件存入用户控制的私有 OSS；只有 OCR、摘要或图片描述所需内容会发送给配置的 Qwen 服务。界面在首次 AI 处理前给出这一事实提示，并使用短期签名 URL 预览私有文件。

## 12. 测试与验收

### 12.1 自动测试

- 契约测试：类型、状态和 API 输入输出。
- 单元测试：文件类型识别、哈希、URL 规范化、SSRF 拦截、状态转换、文本截断。
- 解析器测试：可搜索 PDF、扫描 PDF、DOCX、图片、文字、普通 HTML、PDF URL 和异常文件。
- 服务测试：上传、列表、选择、重试、删除、重复资料、AI 失败回退。
- 前端测试：四种添加入口、状态显示、勾选门槛、错误和重试。
- 端到端测试：上传混合资料，等待解析，勾选资料，确认资料包并进入视觉诊断。

AI 测试默认使用 Mock Gateway，不在常规测试中消耗真实 Qwen 额度；另设手动冒烟测试验证真实 Qwen。

### 12.2 验收场景

1. 上传一个有文字层的 PDF，可查看按页提取内容和 AI 摘要。
2. 上传扫描 PDF，可看到 OCR 内容和失败页提示。
3. 上传 DOCX，可提取段落与表格文字。
4. 上传 PNG/JPG，可看到缩略图、OCR、图片描述和摘要。
5. 粘贴文字，可保存、摘要并选入分析。
6. 提供公开网页链接，可看到正文预览、来源信息和 AI 摘要。
7. 提供内网 URL、登录页、超大文件或旧版 DOC，系统安全拒绝并解释原因。
8. Qwen 不可用时，基础解析仍完成，资料仍可选入分析。
9. 刷新页面或重启服务后，资料和状态仍存在。
10. 没有已勾选的可用资料时，不能进入视觉诊断。

## 13. 明确不做

- 自动全网搜索、Fast Research 和 Deep Research。
- 递归爬取站点、自动点击网页或登录账号。
- 视频、音频、PPT、Excel、Google Drive 和云盘同步。
- 老式 `.doc` 解析。
- 文档协同编辑、全文向量检索和跨项目知识库。
- 把 AI 摘要当作唯一事实来源。

## 14. 后续扩展

统一资料模型为后续网络探索预留 `origin`、来源可信度、推荐理由和研究任务关联字段。Fast Research 或 Deep Research 找到的网页，应先作为候选来源，由用户确认后再进入同一资料池；不另建一套平行来源系统。
