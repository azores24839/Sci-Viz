# Sci AI Studio 资料上传与解析实施计划

**目标：** 在现有 Studio 资料输入节点内实现可持久化的 PDF、DOCX、PNG/JPG、粘贴文字和公开网页资料导入，并提供 OCR、AI 摘要、选择管理与后续 Agent 输入。

## 任务 1：共享契约、持久化与存储适配器

- 在 `@studio/contracts` 定义资料类型、状态、创建请求和 API 响应。
- 在 Server 建立持久资料仓库与任务状态。
- 定义统一对象存储接口，提供本地开发适配器和阿里云 OSS 生产适配器。
- 覆盖列表、读取、选择、删除、去重和状态更新测试。

## 任务 2：解析器

- PDF：文字层提取；文字不足时由 Docker/Poppler 转页并执行 Qwen OCR。
- DOCX：提取段落、列表和表格文本。
- 图片：MIME 校验、私有对象预览、Qwen OCR。
- 文字：规范化并截断到上限。
- 网页：SSRF 防护、重定向复检、正文提取、PDF URL 转 PDF 流程。

## 任务 3：AI 摘要与图片理解

- 新增 DashScope OpenAI 兼容 Gateway，服务端读取 `DASHSCOPE_API_KEY`。
- 文本资料生成带“不确定项”的中文摘要。
- 图片结合原图和 OCR 生成客观描述与摘要。
- 无 key、限流或调用失败时返回 `READY_WITHOUT_SUMMARY`，不阻断基础解析。

## 任务 4：资料 API

- OSS 直传签名、上传确认、粘贴文字、网页链接、列表、详情、选择、重试摘要和删除。
- 上传限制 50 MB，单项目 50 份。
- 安全预览原图；不向前端暴露磁盘路径。

## 任务 4.1：异步 Worker

- 上传确认后写入持久任务，API 快速返回。
- Worker 以有限并发处理解析、OCR 和摘要，失败时指数退避并可人工重试。
- 服务或 Worker 重启后继续处理未完成任务。

## 任务 5：Studio 前端

- 资料节点改为汇总卡，不在画布节点内承载完整上传交互。
- 右侧面板实现四种添加入口、状态列表、详情预览、OCR/原文/摘要、选择、重试与删除。
- 至少一份可用且已选资料后才允许进入视觉诊断。
- 资料分析师请求携带已选资料正文、OCR、描述和来源信息。

## 任务 6：验证

- 运行 workspace 测试、类型检查与构建。
- 使用真实样例验证 PDF、DOCX、图片、文字、网页、OSS 适配器和 AI 失败回退。
- 浏览器检查桌面布局、键盘可达性、错误状态与下一步门槛。
- 构建 Linux Docker 镜像并验证容器内 PDF 转页，不依赖 macOS 工具。
