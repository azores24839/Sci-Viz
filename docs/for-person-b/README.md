## 一、项目是干什么的

我们在做一个"科研影像案例库"——收集国内外高校、期刊、企业发布科研新闻时用的配图/视频，分析这些图是"怎么做的"、"给谁看的"、"用什么风格"。

目前已入库 **3400+** 条案例（主要是静态图片），现在缺两类素材：
1. **微信公众号里的科研配图/视频** — 公众号是中文科研传播主阵地，但完全没采
2. **视频/动图/多媒体案例** — 现有案例 99% 是静图，视频类严重不足

**你的任务就是补这两块。**

## 二、你要干什么

### 任务 1：梳理公众号清单（先做这个）
按学科（生命科学/物理/化学/材料/信息科学/工程/医学/环境/地学/数学）各找 3-5 个有科研视觉价值的公众号。记录：
- 公众号名称、微信号
- 属于什么类型（期刊官号/高校机构/课题组/科普媒体）
- 主要内容是什么（论文封面？实验图？科普插画？）
- 图片质量怎么样（高/中/低）
- 有没有视频内容

输出成一张表就行，格式见下面。

### 任务 2：爬取公众号文章图片和视频
对清单里的每个公众号，爬它的历史文章，把里面的**科研相关配图和视频**提取出来。注意：
- 搜狗微信搜索 (`weixin.sogou.com`) 可以搜到历史文章，但反爬比较厉害
- 公众号文章里的图片通常有 `data-src` 属性（懒加载），真实地址在 `data-src` 而非 `src`
- 视频嵌入在 `iframe` 里，来自 `mp.weixin.qq.com`

### 任务 3：从其他平台补充视频案例
B站、视频号、机构官网里的科研科普视频，先手工收集也可以。

## 三、打包给我的数据格式（很重要！）

### 3.1 每一条案例的 JSON 格式

不管你用什么方式采集，最终每条案例按下面这个结构输出一个 JSON 文件：

```json
{
  "sourceUrl": "https://mp.weixin.qq.com/s/xxxxx",
  "sourceDomain": "mp.weixin.qq.com",
  "pageTitle": "文章标题",
  "caseTitle": "这张图片/视频的简短描述（10字以内）",
  "imageUrl": "图片/视频的原始URL",
  "captureType": "image",
  "contextText": "图片/视频周围200字的上下文文字（用于AI分析）",
  "discipline": "生命科学",
  "mediaType": "摄影",
  "contentType": "实验数据",
  "distributionMedium": "静图",
  "notes": "公众号:Nature Portfolio; 文章日期:2025-03-15;"
}
```

### 3.2 字段填写指南

| 字段 | 必填 | 填什么 | 选项参考 |
|------|------|--------|----------|
| `sourceUrl` | 是 | 原始文章链接 | — |
| `sourceDomain` | 是 | `mp.weixin.qq.com`（公众号）或视频站域名 | — |
| `pageTitle` | 是 | 文章标题 | — |
| `caseTitle` | 否 | 图片简短描述 | — |
| `imageUrl` | 是 | 图片直链地址 | — |
| `captureType` | 是 | `"image"` 或 `"video"` | — |
| `contextText` | 否 | 周围文字，帮助AI理解上下文 | — |
| `discipline` | 建议填 | 学科 | `生命科学` `物理` `化学` `材料` `信息科学` `工程` `医学` `环境科学` `地球科学` `数学` |
| `mediaType` | 建议填 | 呈现方式 | `摄影` `3D渲染` `信息图` `插画` `图表` `示意图` `显微成像` `数据可视化` `复合图` `公式` `地图` `截图` |
| `contentType` | 建议填 | 内容对象 | `实验场景` `人物` `设备` `样本` `数据图表` `模型` `自然景观` `微观结构` `概念图` `流程图` `地图` `代码` `公式` `海报` `封面` |
| `distributionMedium` | 建议填 | 媒介形态 | `静图` `动图` `视频` `图组` `交互` `实体` |
| `notes` | 建议填 | 用分号分隔的备注，记录公众号名称、发布日期等 | — |

### 3.3 文件命名和打包方式

```
采集批次命名：  wechat-batch-01-2025-06-03/
                wechat-batch-02-2025-06-04/
                或按学科： wechat-生命科学-2025-06-03/

每条案例一个JSON文件：  wechat-batch-01/case-001.json
                        wechat-batch-01/case-002.json
                        ...

图片文件（可选）：    wechat-batch-01/images/case-001.jpg
                      wechat-batch-01/images/case-002.jpg

最终打包：  wechat-batch-01.zip
            wechat-batch-01-with-images.zip（如果带图片）
```

**命名规则：**
- 批次号从 01 开始，递增
- 日期用 `YYYY-MM-DD` 格式
- JSON 文件里 `imageUrl` 优先填原始网络地址，图片文件作为备份

### 3.4 如果有视频怎么标

```json
{
  "captureType": "video",
  "distributionMedium": "视频",
  "imageUrl": "https://example.com/video-thumbnail.jpg 或视频页面链接",
  "notes": "视频平台:B站; 时长:5:30; BV号:BV1xx411c7mD;"
}
```

视频案例暂不要求下载文件本身，**记录链接 + 缩略图 + 元数据就行**。
